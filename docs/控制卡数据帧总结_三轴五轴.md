# 控制卡数据帧总结（三轴/五轴）

## 1. 约定与基础格式

- 本文基于以下实现整理：
  - `references/DataBuffer.cs`
  - `references/DataGenerator.cs`
  - `core/executor/data_buffer.cpp`
  - `core/executor/data_generator.cpp`（当前为精简版，部分函数待补全）
- 统一约定：
  - `ly` 结尾函数：按三轴用法理解，`A=0, B=0`。
  - 非 `ly` 函数：按五轴用法理解（可带 `A/B`），若未特别指定可先默认 `A=0, B=0`。
- 基础帧长度：每条指令固定 `16` 字节（`8` 个 `uint16`，小端）。
- `addData(arg1..arg8)` 的物理字段顺序固定为：`B, A, Z, Y, X, OPCODE, arg7, arg8`。
- 轨迹离散时间步长：`10 us`（`t = 0.00001 s`）。

## 2. 原子数据帧（最核心）

| 操作 | 代码入口 | 帧内容（arg1..arg8） | 说明 |
|---|---|---|---|
| Mark点（出光） | `addProcessData(X,Y,Z,A,B)` | `(B, A, Z, Y, X, 0x00FF, 0, 0)` | 加工点 |
| Jump点（关光） | `addProcessJumpData(X,Y,Z,A,B)` | `(B, A, Z, Y, X, 0x0000, 0, 0)` | 跳转点 |
| 加工开始标记 | `addProcessBegin()` | `(0,0,0,0,0,0xFF00,0,0)` | 段开始 |
| 加工结束标记 | `addProcessEnd()` | `(0,0,0,0,0,0x1100,0,0)` | 段结束 |
| 频率设置开始 | `setFreqData(freq)` | `(0x00AA,0,0,0,0,0xAA00,0,0)` | 频率命令头 |
| 频率值帧 | `setFreqData(freq)` | `(cnt_low16, cnt_high16, 0,0,0,0,0,0)` | `cnt = 50000/freq` |
| 频率设置结束 | `setFreqData(freq)` | `(0x00AA,0,0,0,0,0x5500,0,0)` | 频率命令尾 |
| 功率设置帧 | `setPowerData(power)` | `(p,0,0,0,0,0xBB00,0,11451)` | 重复10次后 `forceFill()` |

补充：

- `handleBegin()`（在 `addProcessBegin/setFreqData` 内部触发）会预热两包：  
  `FF00 -> Jump(0x8000,0x8000,0x8000,0x8000,0x8000) -> 1100 -> forceFill`，循环两次。
- `forceFill()`：把当前 buffer 余下字节补零到 `1.6MB` 后入读队列，供 TCP 发送。

## 3. 延时是如何落帧的

`genDelayData(X,Y,Z,A,B,delay,delayOn)` 的帧序列：

1. 先做 `correctionly(ref X, ref Y, ref Z)`；
2. 连续写入 `delayOn / 10` 个 Mark 点帧；
3. 再写入 `(delay - delayOn) / 10` 个 Jump 点帧。

也就是“同一坐标点上，用 Mark/Jump 点数量来合成延时”。

## 4. 五轴（非 ly）常用函数与帧模板

## 4.1 轨迹基元

- `genLineDataNew(...)`  
  插补线段，按 `Laser_Switch` 决定是 Mark 还是 Jump；若 `Laser_Switch=1`，前 `LASER_ON_DELAY` 时间用 Jump，之后 Mark。
- `genLineDataNoDelay(...)`  
  插补线段，全程 Mark 点帧（无起光延时切换）。
- `genCircleNew(...)`  
  圆轨迹插补，A/B 锥度随角度变化，按 `LASER_ON_DELAY` 从 Jump 过渡到 Mark。
- `genRectangleData(...)`  
  由多段 `genLineDataNew / genLineDataNoDelay + genDelayData` 拼接矩形回路。
- `genEllipseData(...)`  
  椭圆离散点插补，按点序号与 `LASER_ON_DELAY` 切换 Jump/Mark。

## 4.2 复合加工（带 Begin/End）

- `genFilledRectangleData(...)`
- `genCircleData(...)`
- `genFilledEllipseData(...)`
- `genProcessLineMoveZ(...)`
- `genProcessLineGoAndBack(...)`
- `genProcessLineGoAndBack5D(...)`

这些函数的典型帧模板基本都是：

1. `addProcessBegin`；
2. 一段或多段 Jump 到起点（`genLineDataNew(... Laser_Switch=0 ...)`）；
3. `genDelayData(... JUMP_DELAY ...)`；
4. 主循环：Mark段 + MarkDelay + Jump段 + JumpDelay；
5. 回原点 Jump；
6. `addProcessEnd`。

## 5. 三轴（ly）函数与帧模板（A/B 固定为0）

按你的约定，以下 `ly` 结尾函数按三轴理解，内部落帧时 `A=0,B=0`：

- `genLineDataNewly(...)`
- `genCircleNewly(...)`
- `gen3DCircle_LineDataly_convex(...)`
- `gen3DCircle_LineDataly_convex_filled(...)`
- `genCircle_filledly(...)`
- `genCircleFilledDataly(...)`
- `correctionly(...)`（三轴坐标校正）

三轴帧与五轴帧的本质区别是 A/B 通道恒零，即：

- Mark点：`(0, 0, Z, Y, X, 0x00FF, 0, 0)`
- Jump点：`(0, 0, Z, Y, X, 0x0000, 0, 0)`

## 6. 典型操作的“可直接对照”帧序列

## 6.1 直线往返（五轴/三轴同理）

`Begin -> Jump到首线起点 -> JumpDelay -> Mark正向线 -> MarkDelay -> Jump换行 -> JumpDelay -> Mark反向线 -> ... -> Jump回原点 -> End`

## 6.2 同心圆填充

`Begin -> Jump到最内圈起点 -> JumpDelay -> Mark圆 -> MarkDelay -> Jump到下一圈 -> JumpDelay -> ... -> Jump回中心/原点 -> End`

## 6.3 频率与功率

- 频率：`AA00头帧 -> 计数值帧 -> 5500尾帧`（常配合 `forceFill` 立即下发）。
- 功率：`0xBB00` 功率帧重复10次后 `forceFill`。

## 7. 与当前 C++ 版本对应关系
