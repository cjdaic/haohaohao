## 实现思路

### 一、SVG图案映射到曲面的完整流程

#### 1.1 完整数据流

```
SVG文件 
  ↓
SVG解析器（提取路径）
  ↓
SVG路径点（2D坐标）
  ↓
坐标归一化/映射到UV空间 [0,1]
  ↓
UV路径点序列
  ↓
UV参数化（已有UVMapper）
  ↓
XYZ路径点（3D曲面坐标）
  ↓
路径分段（mark/jump）
  ↓
DataGenerator::genLineData()
  ↓
DataBuffer（uint16_t格式）
  ↓
硬件执行
```

#### 1.2 前处理选项：选择模型上表面

**方案A：使用工作平面裁剪（推荐）**
- 利用已有的"选择加工平面"功能
- 用户通过菜单"工具" → "选择加工平面"
- 设置平面高度，保留平面上方的模型部分
- 优点：已实现，直接可用

**推荐实现步骤：**
1. 加载 model1111.stl
2. 使用"选择加工平面"功能，设置平面高度为模型底部Z值
3. 保留平面上方的部分（即上表面）
4. 对裁剪后的模型进行UV参数化

#### 1.3 SVG路径到UV坐标的映射

**关键问题：**
- SVG使用屏幕坐标系（左上角为原点，Y向下）
- UV空间使用纹理坐标系（左下角为原点，V向上）
- 需要坐标转换和缩放

**映射策略：**
1. **解析SVG边界框**：获取SVG的viewBox或实际边界
2. **归一化到[0,1]**：将SVG坐标归一化
3. **Y轴翻转**：`v = 1.0 - svg_y`（从屏幕坐标转为纹理坐标）
4. **用户可调整**：允许用户指定SVG在UV空间的位置和缩放

### 二、SVG格式支持实现方案

#### 2.1 SVG格式可行性分析

**✅ SVG格式优势：**
- 矢量图形，可无损缩放
- 支持路径（Path）、圆、矩形等基本图元
- 文本格式，易于解析
- 广泛支持，工具丰富（Inkscape、Illustrator等）

**⚠️ 需要考虑的问题：**
- SVG是2D图形，需要映射到UV空间
- 需要解析SVG路径数据（M, L, C, Q等命令）
- 需要处理SVG的坐标系和单位
- 曲线需要离散化为折线

#### 2.2 实现方案

**方案A：使用nanosvg库（推荐）**
- 轻量级，单头文件，易于集成
- 支持基本路径解析
- 适合简单到中等复杂度的SVG

**方案B：使用librsvg**
- 功能完整，支持复杂SVG特性
- 需要更多依赖，但更强大

**推荐：使用nanosvg（简单、快速）**

#### 2.3 SVG路径处理细节

**SVG路径命令：**
- `M x y`：移动到点（MoveTo）
- `L x y`：画线到点（LineTo）
- `C x1 y1 x2 y2 x y`：三次贝塞尔曲线（Cubic Bezier）
- `Q x1 y1 x y`：二次贝塞尔曲线（Quadratic Bezier）
- `Z`：闭合路径（ClosePath）

**处理策略：**
1. **曲线离散化**：将贝塞尔曲线转换为折线（根据精度参数）
2. **路径提取**：提取所有路径点
3. **坐标转换**：SVG坐标 → UV坐标
4. **路径分段**：识别路径断开点，添加jump段

#### 2.4 XYZ路径转换为DataBuffer格式

**关键实现：**

1. **使用DataGenerator::genLineData()**
   - 将连续的XYZ路径点转换为线段
   - 自动处理激光开关（mark/jump）
   - 根据速度参数生成数据点

2. **路径分段逻辑：**
   ```cpp
   // 伪代码
   for (每个路径段) {
       if (路径断开) {
           // 添加jump点
           buffer.addProcessJumpData(X, Y, Z, A, B);
       } else {
           // 使用genLineData生成线段
           DataGenerator::genLineData(
               speed, times,
               X1, Y1, Z1, A1, B1,
               X2, Y2, Z2, A2, B2,
               buffer
           );
       }
   }
   ```

3. **五轴角度计算（A, B）：**
   - 根据曲面法向量计算A、B角度
   - 或使用固定值（如果不需要五轴）

### SVG解析示例（使用nanosvg）
```cpp
#include "nanosvg.h"
#include "nanosvgrast.h"

// 加载SVG
NSVGimage* image = nsvgParseFromFile("pattern.svg", "px", 96.0f);
if (!image) return;

// 遍历所有形状
for (NSVGshape* shape = image->shapes; shape != NULL; shape = shape->next) {
    // 遍历路径
    for (NSVGpath* path = shape->paths; path != NULL; path = path->next) {
        // 提取路径点
        for (int i = 0; i < path->npts - 1; i += 3) {
            float* p = &path->pts[i * 2];
            // p[0], p[1] 是起点
            // p[2], p[3] 是控制点1
            // p[4], p[5] 是控制点2
            // p[6], p[7] 是终点
            // 离散化贝塞尔曲线...
        }
    }
}
nsvgDelete(image);
```

### XYZ到DataBuffer转换示例
```cpp
// 伪代码
void convertXYZPathToDataBuffer(
    const std::vector<PathPoint>& xyz_path,
    double speed,
    int times,
    DataBuffer& buffer) {
    
    buffer.addProcessBegin();
    
    for (size_t i = 0; i < xyz_path.size() - 1; ++i) {
        const auto& p1 = xyz_path[i];
        const auto& p2 = xyz_path[i + 1];
        
        // 检查是否为跳转（距离过大或激光关闭）
        double dist = sqrt(pow(p2.x - p1.x, 2) + pow(p2.y - p1.y, 2));
        bool is_jump = (dist > JUMP_THRESHOLD) || (p1.laser == 0);
        
        if (is_jump) {
            // 跳转：先移动到起点
            buffer.addProcessJumpData(
                static_cast<uint16_t>(p1.x * 1000),
                static_cast<uint16_t>(p1.y * 1000),
                static_cast<uint16_t>(p1.z * 1000),
                0, 0
            );
        }
        
        // 生成线段数据
        DataGenerator::genLineData(
            speed, times,
            p1.x * 1000, p1.y * 1000, p1.z * 1000, 0, 0,
            p2.x * 1000, p2.y * 1000, p2.z * 1000, 0, 0,
            buffer
        );
    }
    
    buffer.addProcessEnd();
}
```

- **librsvg**：功能完整，但依赖较多

## 注意事项

1. **坐标系统转换**：
   - SVG：屏幕坐标系（左上角为原点，Y向下）
   - UV：纹理坐标系（左下角为原点，V向上）
   - 转换公式：`u = svg_x / svg_width`, `v = 1.0 - svg_y / svg_height`

2. **精度控制**：
   - SVG曲线需要离散化为折线
   - 根据加工精度设置采样密度（建议：0.01mm步长）
   - 贝塞尔曲线离散化步长可调

3. **性能考虑**：
   - 复杂SVG路径可能产生大量点（数千到数万）
   - 考虑路径简化算法（Douglas-Peucker）
   - DataBuffer有大小限制（1.6MB），需要分段处理

4. **数据格式**：
   - DataBuffer使用uint16_t格式（0-65535）
   - 坐标单位：微米（mm * 1000）
   - 需要坐标校正（correction函数）

5. **路径分段**：
   - 检测路径断开（距离 > 阈值）
   - 自动添加jump段
   - 处理闭合路径（Z命令）

6. **兼容性**：
   - 确保SVG解析支持常用工具（Inkscape、Illustrator等）导出的格式
   - 测试常见SVG特性（路径、圆、矩形、组合等）

7. **五轴角度**：
   - 如果不需要五轴，A、B设为0
   - 如果需要，根据曲面法向量计算
