基于纹理映射的复杂曲面激光标刻系统 项目名 NBCAM 版本0.0.1
开发文档（Architecture & Development Doc）
1. 项目定位与核心思想

课题目标（闭环）：实现从“曲面模型 → 参数化UV → 纹理/图案生成 → 映射回三维 → 生成带工艺参数的路径 → 驱动载于课题组自研板卡的多轴振镜加工”的最小可用闭环，并在此基础上支持按曲率/几何特征动态调参（P/f/v 等）与工艺策略扩展。

核心方法论（你的研究亮点）
把复杂曲面加工问题转化为纹理空间（UV）里的“二维图案加工问题”：

对曲面网格做 UV 参数化（ABF / ARAP / Boundary-first-flattening 等可切换）。

在 UV 平面上 生成加工图案：轮廓、栅格、环形、纹理图像(导入.svg文件)等。

把 UV 采样点 映射回三维 XYZ，并附加工艺参数（P/f/v、激光开关、跳线等）。

将路径下发给 师兄板卡（经 TCP/gRPC 服务转发）或者RTC-6，完成标刻，板卡数据输入参考。

2. 系统总体架构
2.1 分层结构（推荐）

UI 层（Qt）

模型视图（3D基于vtk）/ UV 视图（2D类似qcad）/ 路径预览（叠加）

工艺参数面板（全局 + 分段 + 按规则自动生成）

作业队列/状态监控/日志

几何与算法层（C++）(CGAL)

网格导入/预处理（OBJ/STL，修复、法向、尺度、分区等）(CGAL)

UV 参数化（可插拔）

图案与填充策略（可插拔）

UV→XYZ 映射与重采样

工艺参数分配（按曲率/能量/图层）

路径后处理（分段、跳线、速度/功率/频率点位更新策略）

任务/通信层（可选同进程或独立进程）

Job 序列化：laser_job.json（可升级 Protobuf）

本地 TCP/gRPC 服务：UI 内置服务或外置服务均可

下位机/驱动： DataBuffer 发送层 参考DataBuffer.cs/DataGenerator.cs,需移植成C++插入项目中

硬件执行层（师兄板卡+杰普特M8激光器+振镜）

Calibration Layer (标定层)： 负责计算 模型坐标 ->振镜坐标 的变换矩阵 + 模型基准面(只加工基准面以上的区域)设定

振镜运动、激光开关、功率/频率/速度实时更新（按设备能力选择“段级更新”或“点级更新”）

3. 关键工作流（从导入到加工）

Import：加载 OBJ/STL → 生成三角网格（CGAL）

Preprocess：修复/去噪/法向统一/缩放/坐标系对齐（与振镜工作坐标一致）

UV Parameterization：选择算法（ABF/ARAP/BFF）→ 得到每个顶点 UV

Pattern in UV：在 UV 平面生成图案

基本图元：线/圆/矩形/椭圆/多边形

填充：线往复/栅格/等距偏置/环形/纹理图像采样

Map UV→XYZ：对 UV 路径点定位到三角面并插值回 XYZ

Assign Process Params：为路径点/路径段赋值工艺参数

全局参数：基准 P/f/v、重复次数、延时等

自适应参数：按曲率/高度/入射角/能量模型分配

Postprocess：分段、跳线、拐角减速、参数更新稀疏化

Export/Send：导出 laser_job.json 或 Protobuf → 本地服务/驱动下发 → 加工

Verify：仿真预览（路径 + 速度/功率图层）→ 统计（长度、时间估计、能量分布）

4. 数据模型规范（laser_job.json）
4.1 设计原则

能表达“加工意图”：图案/填充/策略 + 参数 + 路径

能表达“硬件执行”：跳线/激光开关/分段/时序

可扩展：未来替换为 Protobuf/二进制流，不改上层逻辑

加工模型大小约2000kb以内

5. 模块划分与接口（可插拔设计）
5.1 模块清单

MeshIO：OBJ/STL 导入、网格构建、法向/拓扑检查

Param：参数化算法接口 + 多实现（ABF/ARAP/BFF）

UVPattern：UV 平面图案生成（图元、布尔、偏置、采样）

Mapper：UV→XYZ（面定位、插值、边界处理、岛屿切换）

ProcessField：工艺场生成（曲率/高度/能量模型 → P/f/v）

Planner：路径后处理（分段、跳线、减速、采样密度、排序）

Serializer：JSON/Proto 编解码

Executor：1师兄板卡:DataBuffer(我提供的) 
2 rtc-6板卡（外部服务）

Sim：可视化仿真（3D/UV/热力图/参数图层）

App：Qt UI、任务队列、日志、配置

5.2 插件点（你未来工作最省心的地方）

8. 关键工程难点与对应策略

UV 岛与缝（seam）处理

路径跨岛：强制切段 + jump；或在 UV 生成阶段限制在单岛

UV→XYZ 定位鲁棒性

使用 “点落在哪个三角面” 的加速结构（BVH/AABB tree）

对边界/数值误差做容错（投影 + 最近面回退）

参数实时更新的硬件限制

点级更新不一定可行 → 采用“段级更新 + 关键点更新”

振镜几何标定

明确 transform_model_to_machine 的来源（标定板/标定点拟合）

必须支持：缩放、旋转、平移、镜像、畸变补偿开关
