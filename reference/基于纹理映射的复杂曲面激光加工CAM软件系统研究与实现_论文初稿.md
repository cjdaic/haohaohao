## 摘  要

复杂曲面结构广泛存在于航空航天、能源装备、精密模具、生物仿生和光学器件等领域。随着激光加工技术向高精度、高柔性和复杂三维曲面加工方向发展，如何将二维图案准确映射到复杂曲面，并生成可下发至振镜或控制板卡的加工路径，成为复杂曲面激光标刻与加工软件系统中的关键问题。传统平面标刻软件主要面向二维平面或简单曲面，难以同时满足三维模型处理、局部曲面参数化、图案纹理贴附、曲面路径生成、加工预览和硬件数据帧下发等需求。因此，研究一套面向复杂曲面激光加工的 CAM 软件系统具有明确的工程意义。

本文围绕复杂曲面激光加工中的“图案到曲面”映射问题，设计并实现了一套基于纹理映射的复杂曲面激光加工 CAM 软件系统 NBCAM。系统采用 C++17、Qt6、VTK、CGAL、nanosvg 和 spdlog 等技术实现，形成了由图形交互层、算法控制层、核心算法层和执行通信层组成的软件架构。系统支持 STL/OBJ 三维模型导入、Patch 聚类与选择、曲面局部参数化、SVG 图案纹理贴附、纹理变换编辑、UV 域路径生成、UV 到三维 XYZ 路径映射、路径预览、模拟加工、手动点动、加工平面校正以及 TCP 数据帧下发等功能。

在算法方面，本文将复杂曲面加工区域局部展开到 UV 平面，将 SVG 图案作为纹理映射到选定 Patch 的 UV 域中，并基于纹理覆盖区域生成轮廓和填充路径。针对参数化展开导致的局部拉伸问题，系统引入逆向预扭曲补偿参数，为后续加工精度优化提供接口。在路径映射阶段，系统通过 UV 空间三角面查找和重心插值，将 UV 加工路径反映射为曲面三维路径点，并组织为统一的 LaserJob 加工任务模型。在执行方面，系统实现了默认板卡 TCP 数据帧下发链路，采用 16 字节固定帧格式和 DataBuffer 双缓冲机制，并增加了 Y/Z 轴互换、开关光控制、手动操作、3x3 正方形加工平面校正和 frame 日志导出等联调功能。

系统测试结果表明，所设计的软件系统能够完成从复杂曲面模型导入、曲面区域选择、SVG 纹理映射、曲面路径生成、路径预览到数据帧下发的完整流程。当前系统已具备复杂曲面激光标刻/加工的软件闭环和硬件联调基础，为后续曲面误差校正、变参数加工、加工质量评价和实际样件实验提供了可扩展平台。

## Abstract

Complex surfaces are widely used in aerospace, energy equipment, precision molds, biomimetic structures, and optical devices. With the development of laser processing toward high precision, high flexibility, and three-dimensional surface fabrication, mapping two-dimensional patterns onto complex surfaces and generating executable toolpaths for galvanometer or board controllers have become key issues in CAM software for laser processing. Conventional planar marking software is mainly designed for two-dimensional planes or simple surfaces, and it is difficult to meet the requirements of three-dimensional model processing, local surface parameterization, pattern texture attachment, curved-surface path generation, machining preview, and hardware data-frame delivery at the same time.

This thesis designs and implements a texture-mapping-based CAM software system named NBCAM for laser processing on complex surfaces. The system is developed using C++17, Qt6, VTK, CGAL, nanosvg, and spdlog. A layered architecture is built, including the graphical interaction layer, application control layer, core algorithm layer, and execution communication layer. The software supports STL/OBJ model loading, patch clustering and selection, local surface parameterization, SVG texture mapping, texture transformation editing, UV-domain path generation, UV-to-XYZ path mapping, path preview, simulated machining, manual jogging, processing-plane calibration, and TCP data-frame transmission.

In terms of algorithms, the selected surface patch is parameterized into a UV domain, and the SVG pattern is mapped onto the patch as a texture. Contour and filling paths are generated within the mapped texture region. To reduce the influence of local stretching caused by parameterization, an inverse-stretch prewarp interface is introduced. During path mapping, UV path points are mapped back to three-dimensional surface points through triangle search and barycentric interpolation, and then organized into a unified LaserJob task model. For execution, the system implements a TCP-based data-frame delivery path with a 16-byte frame format and a DataBuffer double-buffer mechanism. Additional engineering functions such as Y/Z axis swapping, laser-output selection, manual operation, 3x3 square calibration, and frame-log export are also implemented.

The system verification shows that the proposed software can complete the whole workflow from complex surface model loading, surface region selection, SVG texture mapping, curved-surface path generation, path preview, to data-frame delivery. The current system provides a software and hardware integration foundation for further research on surface error calibration, variable-parameter processing, machining quality evaluation, and practical sample experiments.

Key words: Computer Aided Manufacturing; Laser Processing; Complex Surface; Texture Mapping; Surface Parameterization; Galvanometer Control

# 1 绪论

## 1.1 研究背景与意义

复杂曲面零件广泛存在于先进制造领域。例如航空航天结构件、发动机叶片、光学自由曲面、精密模具和仿生功能表面等，都具有明显的三维曲面特征。为了获得特殊的光学、摩擦学、电磁学或流体力学性能，常需要在曲面表面制备图案化结构、微沟槽、周期纹理或标识图形。激光加工由于具有非接触、热影响区域可控、加工路径柔性高、适用材料范围广等优点，已经成为复杂曲面表面微结构制备和图案加工的重要方法。

在平面加工场景中，二维图案与加工平面之间的映射关系较为简单，常规标刻软件可直接将 DXF、SVG 或位图图案转换为平面轨迹。但是在复杂曲面加工中，图案设计通常仍来自二维平面，而实际加工对象为三维曲面。二维图案如何贴附到曲面上，曲面局部变形如何控制，二维路径如何转换为三维加工路径，以及路径如何进一步转换为板卡可识别的数据帧，是复杂曲面激光加工软件必须解决的问题。

另一方面，三维激光加工系统涉及多类工程约束。首先，曲面模型通常以 STL 或 OBJ 三角网格形式输入，需要完成网格导入、显示、区域识别和局部参数化。其次，图案加工需要在曲面上生成 Mark 段和 Jump 段，并保留激光开关、速度、功率、频率和延时等工艺参数。再次，真实下发时需要考虑板卡通信协议、数据帧格式、软限位、坐标轴方向、激光开关光和异常处理。仅依赖单一算法难以支撑完整加工流程，因此有必要构建面向复杂曲面激光加工的完整 CAM 软件系统。

本文所研究的 NBCAM 系统以复杂曲面图案加工为目标，围绕“二维 SVG 图案 - 曲面纹理映射 - 曲面加工路径 - 板卡数据帧”的主线展开研究和实现。通过该系统，可以将用户输入的二维 SVG 图案映射到指定曲面 Patch，并生成可预览、可模拟、可下发的三维曲面加工路径，为后续真实样件加工和工艺优化提供基础平台。

## 1.2 国内外研究现状

### 1.2.1 复杂曲面激光加工技术研究现状

复杂曲面激光加工主要包括投影式加工、五轴机床协同加工、动态聚焦振镜加工以及多轴联动加工等技术路线。投影式加工方法通过将二维图案投影到曲面上获得加工轨迹，适合起伏较小或焦深范围内的曲面，但当曲面高度变化较大时容易出现离焦和图案变形。五轴机床协同加工可以通过调整工件姿态使局部曲面接近法向入射，适用于大尺寸复杂曲面，但控制系统复杂、路径规划难度较高。动态聚焦振镜系统能够在一定空间范围内快速调节焦点位置，适合中小范围内的高效率曲面加工。

从软件角度看，复杂曲面激光加工不仅需要几何路径规划，还需要与硬件系统建立稳定的数据交互。加工软件需要输出振镜控制点、激光开关状态、速度、功率和延时等信息，并对硬件通信和安全联锁进行管理。现有通用 CAD/CAM 软件往往对刀具切削路径支持较好，但对振镜激光加工的数据帧、激光时序和三维纹理映射支持不足。因此，面向具体激光加工平台开发专用 CAM 软件仍具有必要性。

### 1.2.2 曲面参数化与纹理映射研究现状

曲面参数化是将三维曲面映射到二维平面的一类方法，是计算机图形学和几何处理中的基础问题。常见参数化方法包括 Tutte 参数化、最小二乘共形映射 LSCM、尽可能刚性映射 ARAP、面积保持参数化等。参数化的目标通常是在保持角度、面积或局部长度关系之间取得平衡。

纹理映射最初主要用于计算机图形学显示，即将二维图像贴附到三维模型表面。其基本思想是为三维模型顶点建立 UV 坐标，通过纹理坐标在二维图像中采样颜色并显示在三维表面上。本文将这一思想扩展到激光加工路径生成中：二维 SVG 图案不再只是显示纹理，而是作为加工区域和加工轨迹生成的依据。也就是说，纹理映射在本文系统中同时承担“可视化贴附”和“工艺路径定义”两种作用。

在曲面加工中，参数化不可避免地带来局部拉伸和压缩。若直接在 UV 平面生成等间距路径，映射回三维曲面后可能出现局部间距不均、图案变形或边界偏差。因此，需要在参数化、纹理变换、路径生成和曲面反映射之间建立一致的坐标关系，并通过补偿方法减小局部变形对加工结果的影响。

### 1.2.3 曲面路径生成研究现状

曲面路径生成通常包括截平面法、投影法、参数化法和偏置轮廓法等。截平面法通过一系列平面与曲面相交得到曲面路径，适合生成规则扫描线，但图案区域表达较复杂。投影法将平面路径按方向投影到曲面上，实现简单，但受曲面起伏、遮挡和投影方向影响。参数化法先将曲面展开到平面，在平面上进行二维路径规划，再映射回曲面，适合与二维图案和纹理区域结合。偏置轮廓法常用于轮廓加工和环形填充，通过对边界多边形进行内外偏置获得加工路径。

本文所实现的系统采用以参数化法为核心的路径生成方式。对于选定的曲面 Patch，系统先生成 UV 域图案区域，再在该区域内生成轴向往返、圆弧往返、回型填充和轮廓路径，最后将 UV 路径映射回三维曲面。这种方法能够较好地复用二维图案处理和路径规划思想，也便于在 3D 视图和 UV 视图中同时验证路径正确性。

### 1.2.4 CAM 软件系统研究现状

CAM 软件系统通常需要包含模型导入、几何处理、路径规划、后置处理、仿真预览和设备通信等模块。对于复杂曲面激光加工而言，软件系统还需要处理纹理图案、局部曲面展开、激光工艺参数、振镜数据帧和硬件联调等内容。现有通用软件往往缺少针对具体振镜控制协议的数据帧下发能力，而单独的标刻控制软件又缺少复杂曲面处理能力。

因此，本文从工程实现角度出发，构建了一个专用 CAM 软件系统。该系统并不只实现某一个算法，而是将曲面参数化、纹理映射、路径生成、任务建模、仿真预览、通信测试和硬件数据帧下发整合到统一流程中，使复杂曲面激光加工从模型到设备具备完整闭环。

## 1.3 复杂曲面激光加工软件存在的问题

结合复杂曲面激光加工需求和当前系统开发过程，现有方案主要存在以下问题。

第一，二维图案与三维曲面之间缺少统一映射模型。传统标刻路径通常在平面坐标系中生成，无法直接表达曲面上的图案变形、边界限制和局部路径间距。若直接将二维路径投影到曲面上，容易受到曲面起伏和坐标方向影响。

第二，曲面加工路径难以同时兼顾显示、规划和执行。路径在显示阶段需要可视化，在规划阶段需要保持拓扑和段属性，在执行阶段又需要转换为硬件数据帧。若各阶段使用不同数据结构，容易出现路径显示正确但发送数据不一致的问题。

第三，硬件坐标轴和模型坐标轴可能不一致。实际联调中，模型坐标轴与振镜通道之间可能存在 Y/Z 轴互换等问题。如果软件没有提供统一的轴映射开关，手动点动、真实加工和日志中的坐标就可能互相不一致。

第四，数据帧下发需要稳定的缓冲机制和反馈信息。振镜板卡通常要求按固定帧格式接收数据，且 TCP 通信可能具有握手、重试和超时等特性。软件既要连续下发加工数据，又不能因等待反馈而阻塞主循环或造成界面与硬件状态脱节。

第五，真实加工前缺少直观校正工具。复杂路径不适合直接用于首次联调。系统需要提供小范围、规则化、可测量的校正图案，以验证坐标方向、比例、中心位置和激光开关状态。

## 1.4 本文研究内容与技术路线

针对上述问题，本文开展了基于纹理映射的复杂曲面激光加工 CAM 软件系统研究。主要研究内容包括：

1. 设计复杂曲面激光加工 CAM 软件总体架构，建立图形界面、算法控制、核心算法和执行通信的分层结构。
2. 实现三维模型导入、Patch 聚类、局部参数化和 UV 视图显示，为纹理映射和路径生成建立曲面参数域。

# 2 复杂曲面激光加工 CAM 软件系统设计

## 2.1 系统需求分析

本文系统面向复杂曲面激光标刻与加工应用。根据加工流程和实际联调需求，系统需要满足以下功能需求。

第一，模型处理需求。系统应支持 STL/OBJ 等三维模型文件导入，能够显示模型、选择曲面区域、执行 Patch 聚类，并支持局部参数化。对于模型尺度不合适或偏离加工区域的情况，系统应提供模型偏置、自动居中和加工区域显示功能。

第二，图案映射需求。系统应支持 SVG 图案导入，并能将图案作为纹理贴附到选定 Patch 上。用户应能够调整纹理缩放、平移、旋转、透明度和补偿参数，并在 3D 视图和 UV 视图中检查贴图位置。

第三，路径规划需求。系统应能在纹理覆盖区域内生成加工路径，包括轮廓路径、填充路径、跳转路径和分层路径等。路径应保留 Mark/Jump 类型、激光状态、加工参数和段 ID，以便后续预览和执行。

第四，执行通信需求。系统应支持将加工路径转换为板卡可识别的数据帧，支持 TCP 连接、超时、重试、双缓冲发送、日志记录和错误提示。对于 RTC-6 板卡，系统应保留相应执行器接口。

第五，联调验证需求。系统应提供模拟加工、路径预览、手动点动、Y/Z 轴互换、校正图案和日志导出功能，使操作者能够在真实出光前验证路径、坐标方向、加工区域和通信状态。

## 2.2 软件总体架构

NBCAM 软件采用分层架构设计，主要包括图形交互层、应用控制层、核心算法层和执行通信层，如图 2-1 所示。

> 图 2-1  NBCAM 软件总体架构图（待补充）

本文系统的软件架构设计遵循“界面交互、算法处理、任务组织和硬件执行相互解耦”的原则。复杂曲面激光加工 CAM 软件同时涉及三维模型显示、曲面参数化、SVG 图案解析、路径规划、工艺参数管理、TCP 数据帧下发和设备联调等多个环节，如果将所有逻辑直接写入界面层，不仅会造成代码耦合严重，也会使后续调试和功能扩展变得困难。因此，系统采用分层架构，将用户操作、业务流程、核心算法和底层执行分别放在不同层次中实现，使每一层只关注自身职责，并通过明确的数据结构进行通信。

系统最上层为图形交互层，主要负责用户界面、三维显示和操作入口。该层基于 Qt 实现主窗口、菜单栏、参数面板、路径预览窗口、配置对话框和状态栏，基于 VTK 实现三维模型、Patch、纹理和加工路径的可视化显示。图形交互层的设计目的，是把复杂的加工流程以直观方式呈现给用户，使用户能够完成模型导入、Patch 选择、参数化、SVG 贴附、参数设置、模拟加工和真实加工等操作。该层不直接承担复杂算法计算，而是将用户指令转交给应用控制层处理，从而避免界面代码与算法细节混杂。

第二层为应用控制层，核心由 `ApplicationController` 负责。该层是界面与核心算法之间的桥梁，承担流程编排和状态管理任务。用户在界面中点击“参数化选中 Patch”“应用 SVG 纹理到 Patch”“执行路径规划”等操作后，控制层会根据当前模型状态、选中 Patch、纹理信息和工艺参数调用相应算法模块，并将结果返回界面显示。应用控制层保存当前模型、参数化结果、SVG 图案、UV 路径、三维路径和 `LaserJob` 加工任务等关键状态，使整个加工流程具有连续性。通过设置这一层，系统可以把“用户想做什么”转换为“算法模块应按什么顺序执行”，提高了业务流程的可维护性。

第三层为核心算法层，主要位于 `core` 目录下，是软件实现复杂曲面加工能力的核心。该层包含模型导入与网格处理、Patch 聚类、曲面参数化、SVG 解析、UV 路径生成、UV 到 XYZ 映射、工艺参数管理和路径规划等模块。其中，网格处理模块负责将 STL/OBJ 模型转换为统一的三角网格结构；参数化模块负责将选中曲面区域展开到二维 UV 域；图案模块负责解析 SVG 并在 UV 域内生成轮廓或填充路径；映射模块负责通过三角形查找和重心坐标插值将 UV 路径反映射为三维曲面路径；路径规划模块负责将路径组织成 Mark 段和 Jump 段。核心算法层不关心具体界面按钮，也不直接处理 TCP 发送，而是专注于从几何模型到加工路径的数据转换。

第四层为执行通信层，主要负责将上层生成的 `LaserJob` 加工任务转换为硬件可识别的数据帧，并完成下发。该层包含默认板卡执行器、RTC-6 执行器接口、TCP 通信、DataBuffer 双缓冲和 frame 日志等模块。默认板卡执行链路按照固定 16 字节帧格式编码坐标、操作码和开关光状态，并结合软限位、Y/Z 轴互换、开关光选项和延时参数生成最终发送数据。DataBuffer 双缓冲用于分离数据帧填充和 TCP 发送，减少通信等待对主流程的影响。执行通信层的设计重点是保证发送到硬件的数据与路径、坐标映射和日志保持一致，便于联调和问题追踪。

从数据流角度看，系统整体流程可以概括为：外部 STL/OBJ 模型首先进入网格处理模块，经过曲面预处理、Patch 聚类和局部参数化后形成 UV 参数域；SVG 图案被解析并映射到选中 Patch 的 UV 区域中，随后在该区域内生成 UV 路径；UV 路径通过映射模块转换为三维曲面路径，并由路径规划模块组织为 `LaserJob`；最后，执行通信层将 `LaserJob` 转换为板卡数据帧并通过 TCP 或其他执行后端下发。该数据流使系统从“模型和图案输入”到“硬件执行输出”形成清晰闭环。

这种架构设计具有较好的扩展性。若后续需要增加新的参数化算法，只需在核心算法层增加相应实现，并保持参数化结果的数据接口一致；若需要增加新的填充策略，只需在图案路径生成模块扩展策略类；若需要更换控制板卡或增加五轴平台执行方式，则可在执行通信层增加新的执行器，而不必重写模型显示和路径生成逻辑。对于复杂曲面激光加工这种软硬件结合系统而言，分层架构不仅提高了代码可维护性，也使算法研究、界面开发和设备联调能够相对独立地推进。

图形交互层位于 `apps/laser_cam_qt`，基于 Qt6 实现主窗口、菜单、对话框、参数面板、路径预览和状态栏；基于 VTK 实现 3D 模型显示、Patch 高亮、纹理显示和路径显示；基于自定义 UVView 实现 UV 坐标、纹理和路径的二维显示。

应用控制层由 `ApplicationController` 负责，是 UI 与核心算法库之间的主要边界。控制器维护当前模型、参数化结果、UV 路径、XYZ 路径和 LaserJob 任务，并封装模型导入、参数化、图案生成、路径映射、工艺参数分配和任务保存等操作。

核心算法层位于 `core` 目录下，包含网格处理、参数化、SVG 解析、填充策略、UV 映射、工艺参数模型、路径规划和任务序列化等模块。各模块之间通过明确的数据结构交互，避免 UI 直接承担复杂算法逻辑。

执行通信层位于 `core/executor`，包括 `BoardExecutor`、`Rtc6Executor`、`DataBuffer`、`TcpSocket` 和 `frame_log_utils` 等。该层负责执行器预检、坐标编码、数据帧生成、TCP 下发和日志导出。

系统目录结构如下：

```text
apps/laser_cam_qt        Qt 主程序、主窗口、视图和交互入口
core/job                 LaserJob 数据模型和 JSON 序列化
core/mesh                网格导入、预处理和 Patch 聚类
core/param               LSCM、ARAP、Authalic 参数化
core/pattern             SVG 解析和填充路径策略
core/mapping             UV 到三维 XYZ 映射
core/process             工艺参数模型
core/planner             路径规划和路径段后处理
core/executor            板卡执行器、TCP、双缓冲和日志
reference                参考资料、协议、开发文档和用户手册
```

## 2.3 图形界面与三维显示设计

系统主界面由 3D 模型视图、UV 视图、纹理列表、参数面板、路径预览和加工控制按钮组成。3D 视图用于显示导入模型、Patch 聚类结果、SVG 纹理、加工路径和已加工高亮；UV 视图用于显示参数化结果、纹理区域和 UV 路径；右侧参数面板用于设置功率、频率、速度、填充间距、填充角度、填充策略、轮廓模式和激光输出状态。

3D 显示采用 VTK 管线实现。模型加载后，系统将三角网格转换为 VTK PolyData 并渲染。Patch 选择后，模型视图可以高亮选中区域。应用 SVG 纹理时，系统将 SVG 渲染为图像数据，构造 `vtkTexture`，再为 Patch 子网格生成纹理坐标数组 `TCoords`，从而将二维图案显示在三维曲面上。

UV 视图用于验证参数化和纹理映射效果。系统可以显示 UV 点、UV 路径、纹理区域和已加工路径段。对于多个 Patch 的路径保存和显示，系统还维护 Patch 的 UV 显示范围，使不同 Patch 的 UV 结果可以在界面中有序布局。

右上角加工控制按钮包括开始、暂停、急停、刷新和模拟加工。模拟加工不下发硬件数据，仅用于观察路径顺序和高亮进度。真实加工时，系统按照当前路径规划结果和右侧开光选项生成数据帧。

## 2.4 数据模型与模块组织

系统使用 `LaserJob` 作为加工任务的统一数据模型。`LaserJob` 包含任务元数据、坐标变换、参数化信息、默认工艺参数和路径段集合。路径段由 `PathSegment` 表示，路径点由 `PathPoint` 表示。

`PathPoint` 中同时保存 UV 坐标和三维 XYZ 坐标，其中 UV 坐标用于纹理和路径来源追溯，XYZ 坐标用于真实加工执行。每个点还包含 A/B 轴值、激光状态和点级工艺参数覆盖。`PathSegment` 则用于区分 Mark 段和 Jump 段，并记录填充策略和段级参数。

这种数据模型使系统能够统一处理三类需求：

1. 可视化需求：路径预览和 3D/UV 高亮可以直接读取段 ID 和点坐标。
2. 算法需求：路径规划和后处理可以按段组织 Mark/Jump 关系。
3. 执行需求：执行器可以将路径点编码为板卡数据帧。

系统中主要模块与数据流关系如下：

```text
TriangleMesh
  -> PatchClusterer
  -> Parameterizer
  -> UVCoord
  -> SVGToUV / PatternStrategy
  -> UVPathPoint
  -> UVMapper
  -> PathPoint
  -> PathPlanner
  -> LaserJob
  -> BoardExecutor / MainWindow MachiningPlan
  -> DataBuffer / TcpSocket
```

## 2.5 主要功能的目的、原理与实现说明

为了使论文正文不仅描述系统“有哪些功能”，还能够说明“为什么需要这些功能、这些功能按什么原理工作、在项目中怎样落地”，本节对 NBCAM 软件中的主要功能做进一步展开。这里所列功能并不是彼此孤立的界面按钮，而是围绕复杂曲面激光加工这一目标形成的完整流程：先完成三维模型理解和局部曲面划分，再把二维图案映射到曲面参数域中，随后生成可执行的加工路径，最后通过通信链路把路径转为板卡或振镜可识别的数据帧，并通过模拟、日志、手动点动和校正功能完成闭环验证。

### 2.5.1 三维模型导入与显示功能

三维模型导入是整个系统的起点，其目的不是简单地把 STL 或 OBJ 文件读进内存，而是建立“加工对象”的几何基础。对于复杂曲面加工来说，模型本身决定了后续 Patch 划分、参数化质量、纹理映射扭曲程度以及加工路径在空间中的分布范围。如果模型无法正确导入，那么后续任何路径规划和数据帧生成都失去意义。

从原理上看，系统首先解析三角网格文件中的顶点、法向和面片索引，并将其转换为统一的内部网格结构。由于 STL 和 OBJ 都以三角面片作为几何基础，系统在读取阶段重点解决三个问题：第一是保持模型拓扑信息完整，确保相邻三角形关系可用于后续 Patch 聚类；第二是对模型单位和尺度进行一致化处理，避免输入模型与机床工作坐标之间出现数量级偏差；第三是将网格转为 VTK 可渲染数据，以便在三维视图中实时显示。

在实现上，模型显示层承担两个职责。一方面，它要把网格、包围盒、坐标轴、Patch 高亮和纹理渲染放在同一视图中展示，帮助用户判断加工范围；另一方面，它还要为选择操作提供交互基础，例如通过拾取机制选中某个 Patch，或者显示当前加工段在模型上的位置。对于操作员来说，这种显示不是“看图”，而是对加工对象是否可加工、加工区是否偏移、选区是否正确的直接确认。

### 2.5.2 Patch 聚类与局部参数化功能

Patch 聚类的目的，是把一个复杂的全局曲面拆分成若干局部上更容易处理的区域。复杂曲面直接整体参数化往往会出现严重拉伸、折叠或边界扭曲，而将模型划分为 Patch 后，每个局部曲面都更接近平滑片段，便于展开到 UV 平面。这一功能实际上是在几何复杂性和算法可控性之间建立平衡。

其基本原理是利用相邻三角面片之间的法向差异和二面角差异来判断局部连续性。当两个面片的法向变化较小、边界连接连续时，系统倾向于将它们归入同一 Patch；当局部几何发生明显转折时，则拆分为不同 Patch。这样做的目的有两个：一是减少参数化失败概率，二是让用户能够按加工区域逐块控制纹理和路径，而不是在整个模型上做一次性展开。

局部参数化则是把三维 Patch 映射为二维 UV 域，供后续纹理贴附和路径生成使用。系统接入 LSCM、ARAP 和 Authalic 等算法，本质上是提供不同的几何保持偏好。LSCM 更注重角度保持，适合图案外形不能明显变形的情况；ARAP 更强调局部刚性，适合希望路径局部长度稳定的场景；Authalic 更偏向面积保持，适合加工区域大小比形状更敏感的任务。项目中保留多种算法，并不是重复实现，而是为了让操作者根据模型曲率和工艺目标选择更合适的展开方式。

### 2.5.3 SVG 图案贴附与纹理管理功能

SVG 图案导入和贴附是本系统区别于普通平面标刻软件的关键能力之一。它的目的在于把二维设计内容作为“纹理对象”附着到复杂曲面上，使图案设计、路径规划和最终加工结果共享同一套空间语义。对于用户而言，这意味着图案不再只是静态文件，而是可以直接在模型表面定位、缩放、旋转和预览的工艺元素。

从原理上看，SVG 具有矢量图特性，适合在不同分辨率下保持轮廓清晰。系统使用 SVG 解析库提取路径、曲线和填充区域，再将其栅格化或转换为几何轮廓，最终映射到 Patch 的 UV 域。这样做的好处是：一方面保留图案边界的几何精度，另一方面可以直接在 UV 域中进行路径生成，而无需先把图案投影到三维空间后再做二次处理。

纹理管理的实现不仅包括“导入文件”，还包括“定义纹理如何和曲面关系变化”。因此系统提供缩放、平移、旋转、透明度和逆向预扭曲等参数。缩放用于控制图案尺寸与加工尺度的对应关系；平移用于控制图案落点；旋转用于适配曲面主方向或工艺扫描方向；透明度主要服务于显示和调图，不直接影响加工；逆向预扭曲则用于补偿参数化过程中产生的几何失真。通过这些参数，用户可以把“图案放在曲面上”这一抽象需求转化成可调、可视、可验证的工程动作。

### 2.5.4 路径规划与 LaserJob 任务组织功能

路径规划是把图案从“可见”变为“可执行”的关键步骤。其目的不是单纯生成点序列，而是构建一个同时满足几何正确性、执行可读性和工艺可解释性的加工任务。复杂曲面加工往往包含轮廓、填充、跳转、分层、段级参数和点级参数等多种信息，如果只是输出一串坐标点，后续模拟加工和数据帧下发都会失去结构化依据。

系统使用 `LaserJob` 作为统一任务容器，其原理是把加工对象、参数化结果、工艺默认值和路径段组织到同一个数据模型中。这样一来，UI 预览、模拟加工、保存/加载、执行器下发都可以读取同一份数据，而不需要为每个模块单独维护一套路径结构。`PathSegment` 用来区分 Mark 和 Jump 两类运动，`PathPoint` 用来保存 UV、XYZ、A/B、激光状态和参数覆盖。通过这种设计，几何路径和执行语义被绑定在一起，避免出现“界面看到一条路，板卡收到另一条路”的问题。

在实现上，路径生成大致经历“UV 轨迹生成 -> UV 到 XYZ 映射 -> 路径段组织 -> 参数绑定 -> 预览/导出 -> 执行”几个阶段。每个阶段都可以独立检查，方便调试。例如若纹理边界有误，问题通常出在 SVG 解析或 UV 生成；若路径方向不对，问题通常出在参数化方向或轴映射；若发送后运动异常，问题通常出在帧编码或通信层。将路径组织为 LaserJob 的一个直接收益，就是这些问题都可以定位到明确层次，而不再混淆在单一函数中。

### 2.5.5 路径预览与模拟加工功能

路径预览和模拟加工的目的，是在真实出光之前建立一个“软件内验证”环节。复杂曲面激光加工一旦进入硬件执行阶段，错误的代价往往高于普通平面标刻，因此在软件中提供预览并不是锦上添花，而是工程安全的一部分。

路径预览的核心原理是把路径段、路径点、起止位置、类型和长度等信息以表格和图形方式表达出来，让用户在发送前确认路径是否符合预期。模拟加工则在此基础上进一步把“当前加工到哪一段”这个概念可视化出来：已加工段高亮，当前段高亮，未加工段保持原状。这样做的意义在于，它把抽象的段级执行过程变成可观察过程，便于发现路径顺序错乱、段断裂、段间跳转过长或局部高密度路径异常等问题。

在实现上，模拟加工并不依赖真实硬件返回，而是按照路径列表的顺序更新 UI 状态。因此它更接近一种“执行脚本回放”机制。这样可以避免通信链路未连通时界面完全失效，也能在没有板卡或振镜的情况下检查路径组织是否合理。对于这类系统而言，模拟加工的价值不只是演示，而是用来验证算法输出是否在逻辑上成立。

### 2.5.6 数据帧生成、双缓冲和通信功能

数据帧生成的目的，是把几何路径转换成板卡或振镜能够直接识别的控制信号。这里的关键不是“有没有发出去”，而是“发出去的内容是否严格对应于路径点、工艺状态和轴映射规则”。因此，通信层需要同时处理坐标编码、帧格式、开关光位、延时参数、日志记录和发送节奏。

从原理上看，系统采用固定长度数据帧，将坐标和操作码编码到若干 16 位字段中。这样做的优势是协议解析简单、发送稳定、对硬件缓存友好。为了避免上层路径规划与下层网络发送互相阻塞，系统使用 DataBuffer 双缓冲。一个缓冲区负责填充待发数据，另一个缓冲区负责等待发送或已经进入发送线程。双缓冲的意义在于把“生成数据帧”和“发送数据帧”解耦，从而提高主循环连续性，减少因通信等待导致的 UI 卡顿。

实现上，发送流程还需要结合 `waitUntilIdle()`、`appendBytes()`、`forceFill()` 等机制。需要强调的是，排空等待只表示发送侧缓冲或队列状态结束，并不等价于振镜物理运动已经完成。因此系统在设计时把“传输完成”“运动完成”和“界面进度完成”分开处理，以避免把通信反馈误判为机械反馈。实际工程中，若把这几者混为一谈，就很容易出现界面显示已经结束，但设备实际上还在运动的错配现象。

### 2.5.7 手动点动与 Y/Z 轴互换功能

手动点动的目的，是给设备联调和坐标验证提供一个最小闭环。对于复杂曲面加工系统来说，自动路径往往很长，错误一旦发生不易定位，而手动点动能够用短距离、单方向的移动迅速验证轴向、零点、限位和输出逻辑是否正确。它在系统中的作用类似“低风险探针”。

其原理并不复杂：用户输入步长后，点击方向按钮，系统按照当前逻辑坐标计算目标位置，并生成对应的数据帧发送给板卡。若目标位置超出软限位范围，则直接拒绝发送或提示错误。STOP 回中按钮则不是简单停止，而是把当前控制目标重设到软限位的中点，以便将设备恢复到一个已知、安全、可再次操作的位置。

Y/Z 轴互换功能之所以单独保留，是因为模型坐标轴、加工平面定义和硬件振镜通道经常不完全一致。对用户来说，这个开关的目的不是“调整显示”，而是统一逻辑坐标与实际硬件通道之间的关系。勾选后，系统在帧编码、日志输出、手动点动和实际加工阶段都使用相同的轴交换规则，以保证“看到的、记下的、发出去的”是同一套坐标语义。

### 2.5.8 加工平面校正功能

加工平面校正的目的，是在正式加工之前确认中心、方向、尺度和轴映射是否正确。对于复杂曲面激光加工系统而言，校正不是附属功能，而是决定后续样件是否可信的前置步骤。若不先做校正，哪怕路径规划完全正确，实际结果也可能因为坐标轴错误、比例偏差或零点漂移而失真。

系统设计了 3x3 正方形阵列校正图案，其原理是用一组结构简单、重复性高、几何关系明确的图形去检查加工平面的全局一致性。中心正方形位于零点，周围八个正方形则覆盖前后左右及四个对角方向。这样一来，用户可以同时观察是否存在坐标翻转、轴向偏置、中心偏移和伸缩不一致。选择正方形而不是复杂图案，是因为正方形边界直观，测量简单，最适合用作初始标定。

在实现上，校正功能把逻辑平面上的正方形转换为一组可发送的路径点，并结合当前开光状态生成 frame 日志。它既可以在不开光状态下做轨迹检查，也可以在开光状态下做实际打标。通过这种分层验证，系统既能检查“轨迹有没有走对”，也能检查“真实加工出来的几何有没有偏差”。

### 2.5.9 CLI 与日志功能

CLI 和日志功能的目的，是让系统在没有完整 GUI 操作的情况下仍能完成基本验证和追踪。CLI 适合批量脚本式操作，比如连续执行模型导入、参数化、图案导入和路径规划；日志则用于记录每次加工或通信过程中的关键状态，帮助开发者回溯问题。

从原理上看，CLI 相当于把 GUI 中的关键业务动作抽象成文本命令。这样可以减少人工点击过程中的偶然性，也方便在调试时复现实验流程。日志则记录了更低层的信息，包括连接状态、帧生成情况、开关光状态、轴映射结果和异常提示。对于这类硬件联调软件，日志不是附加品，而是定位“为什么设备没按预期动作”的最重要依据之一。

### 2.5.10 本节小结

通过对上述功能逐项分析可以看到，NBCAM 的各项功能虽然分布在不同界面和模块中，但它们共同服务于同一条主线：把复杂曲面上的二维图案，可靠地转换为可执行、可预览、可校正、可追踪的数据流。模型导入解决对象来源，Patch 聚类和参数化解决二维化问题，纹理贴附解决图案定位问题，路径规划解决运动轨迹问题，通信与日志解决设备执行问题，手动点动和校正解决联调闭环问题。也正因为这些功能彼此咬合，系统才能从“图案设计”走向“真实加工”。

## 2.6 软件涉及的主要算法

NBCAM 软件并不是单一算法程序，而是由多类算法共同构成的复杂曲面加工流程。其算法链路从三维模型输入开始，经过曲面预处理、Patch 聚类、局部参数化、SVG 图案映射、UV 域路径生成、曲面路径反映射、工艺参数绑定和数据帧编码，最终形成可下发到板卡的加工数据。各类算法在系统中承担不同角色，但它们共同服务于一个目标：将二维图案可靠地转换为复杂曲面上的可执行激光加工轨迹。

### 2.6.1 曲面网格预处理算法

曲面网格预处理算法主要用于整理外部输入模型，使其满足后续几何计算要求。STL/OBJ 模型通常由大量三角面片组成，软件首先读取顶点、面片、法向和索引关系，并构建统一的三角网格数据结构。随后计算模型包围盒、中心点、尺度范围和面片邻接关系。包围盒用于判断模型是否超出加工区域，中心点用于模型自动居中和偏置，邻接关系则为 Patch 聚类和边界分析提供基础。

该算法的核心目的在于把不确定的外部模型转化为稳定的内部几何对象。对于实际加工系统而言，模型不仅要能显示，还要能被参数化、能生成路径、能映射回三维坐标。因此预处理阶段需要尽量识别退化面片、异常尺度、法向不一致和局部网格过稀等问题。对于局部面片过大的区域，系统可通过自适应均匀细分提高曲面离散精度，使后续纹理映射和路径显示更加连续。

### 2.6.2 Patch 聚类算法

Patch 聚类算法用于将复杂曲面划分为多个局部加工区域。复杂曲面整体展开通常会产生较大变形，因此系统先根据面片之间的几何连续性进行区域划分。算法主要依据相邻三角面片的法向夹角、二面角和拓扑邻接关系判断两个面片是否属于同一局部曲面。当相邻面片法向变化较小、曲面过渡较平滑时，将其归为同一 Patch；当存在明显折边或曲率突变时，则形成 Patch 边界。

该算法的意义在于降低参数化难度，并使用户能够按局部曲面区域进行纹理放置和路径规划。Patch 聚类结果不仅影响界面上的选区显示，也直接影响后续 UV 展开的边界形状和纹理映射质量。如果 Patch 过大，参数化变形可能较严重；如果 Patch 过小，图案可能被切割得过碎。因此聚类阈值需要根据模型形状和加工区域进行调整。

### 2.6.3 曲面局部参数化算法

曲面局部参数化算法用于将选中 Patch 从三维空间展开到二维 UV 平面。软件中涉及 LSCM、ARAP 和 Authalic 等参数化方法。LSCM 主要强调角度保持，使局部形状在展开后尽量不发生角度畸变，适合对图案形状要求较高的加工场景；ARAP 追求局部刚性保持，尽量减少局部变形，适合希望长度关系较稳定的曲面；Authalic 侧重面积保持，适合加工覆盖率和区域面积更敏感的任务。

参数化算法的基本思想是为曲面顶点求解对应的二维坐标，使三维网格在二维平面中形成无翻转、尽量低畸变的展开结果。展开后的 UV 坐标是纹理贴附和路径生成的基础。由于一般复杂曲面无法做到无失真展开，因此不同参数化算法实际上是在角度、长度和面积之间做权衡。系统提供多种方法，是为了让用户根据模型曲率和图案需求选择更合适的展开策略。

### 2.6.4 SVG 解析与纹理映射算法

SVG 解析算法用于读取二维图案文件中的矢量几何信息，包括路径、曲线、轮廓、填充区域和画布范围。软件使用 SVG 解析库将图案转换为可显示和可计算的内部数据。一方面，系统将 SVG 栅格化为图像并作为纹理贴附到 Patch 上；另一方面，系统提取图案区域，用于后续 UV 域路径生成。

纹理映射算法的核心是建立 SVG 图案坐标、Patch UV 坐标和三维曲面坐标之间的关系。系统根据参数化结果为 Patch 顶点生成纹理坐标，并通过缩放、平移和旋转等变换控制图案在曲面上的位置。由于 SVG 图像坐标和 VTK 纹理坐标方向不同，映射过程中还需要处理坐标翻转。该算法的目的，是让用户在三维模型上看到的纹理位置与后续加工路径使用的区域保持一致。

### 2.6.5 纹理变换与逆向预扭曲补偿算法

纹理变换算法用于对已贴附的 SVG 图案进行缩放、平移和旋转。其数学形式可以表示为先将 UV 坐标平移到局部中心，再进行尺度变换和旋转变换，最后加上平移量回到目标位置。通过该算法，用户可以在不修改原始 SVG 文件的情况下调整图案尺寸、方向和位置。

逆向预扭曲补偿算法用于减小曲面参数化带来的映射变形。由于 UV 域中的规则图案映射到三维曲面后可能发生拉伸或压缩，系统提供补偿接口，在生成路径前对图案或采样区域进行反向调整。其基本思想是估计 UV 到三维曲面的局部拉伸趋势，并在 UV 域中提前施加相反方向的变形，使最终落到曲面上的图案更接近目标形状。当前系统中该算法主要作为可调接口保留，后续可结合实际加工测量数据进行标定。

### 2.6.6 UV 域路径生成算法

UV 域路径生成算法用于在 SVG 图案覆盖区域内生成加工轨迹。系统支持轴向往返填充、圆弧往返填充、回型填充、轮廓加工和 Z 轴分层填充等策略。轴向往返填充通过一组平行扫描线与图案区域求交，生成覆盖均匀的往返路径；圆弧填充使用圆弧族作为扫描曲线，适合环形或弧形结构；回型填充通过边界逐层偏置形成由外到内或由内到外的路径；轮廓加工直接沿 SVG 边界生成轨迹；Z 轴分层填充则用于具有层次加工需求的场景。

路径生成算法需要处理图案边界、内部孔洞、多区域断开和扫描方向等问题。当填充线穿过多个有效区域时，算法会将其切分为多个加工段，并在段间设置跳转标记。这样，后续路径规划模块就能区分需要开光加工的 Mark 段和需要关光移动的 Jump 段。UV 域路径生成的优势在于计算过程发生在二维平面中，区域求交、填充线生成和轮廓处理都相对简单，之后再统一映射回三维曲面。

### 2.6.7 UV 到三维曲面的反映射算法

UV 到三维曲面的反映射算法用于把二维路径点转换为曲面上的三维坐标。对于每个 UV 路径点，系统首先在参数化后的 UV 三角网格中查找包含该点的三角形，然后计算其重心坐标。由于 UV 三角形的三个顶点与三维网格中的三个顶点一一对应，系统可以使用同一组重心权重对三维顶点进行插值，得到路径点在曲面上的 XYZ 坐标。

该算法的关键在于三角形查找效率和边界鲁棒性。对于大规模模型，若每个路径点都遍历所有三角形，计算效率会明显下降。因此系统可使用 UV 空间网格加速查找，将候选三角形限制在局部网格单元中。若加速查找失败，则回退到线性查找，以避免边界点或浮点误差导致有效路径丢失。反映射算法直接决定路径是否落在正确曲面位置，是连接纹理映射与真实加工的关键算法。

### 2.6.8 路径段组织与工艺参数绑定算法

路径段组织算法用于将离散路径点整理为可执行的 `LaserJob`。系统根据路径断点、激光状态和图案区域关系，将路径划分为 Mark 段和 Jump 段。Mark 段表示激光打开并进行加工，Jump 段表示激光关闭并移动到下一起点。每个路径段包含段 ID、段类型、路径策略、点列表和工艺参数覆盖信息。

工艺参数绑定算法用于为路径点或路径段分配功率、频率、速度、开关光延时等加工参数。系统中参数优先级通常为点级参数、段级参数、任务默认参数和执行器默认参数。通过这种组织方式，软件可以支持统一参数加工，也可以扩展到曲率相关的变参数加工。路径段组织和工艺参数绑定使系统不再只是生成坐标点，而是生成具有执行语义的加工任务。

### 2.6.9 数据帧编码与双缓冲发送算法

数据帧编码算法用于将 `LaserJob` 中的路径点转换为板卡协议要求的固定格式数据。默认板卡链路采用 16 字节帧，每帧包含 `B,A,Z,Y,X,OPCODE,arg7,arg8` 等字段。系统需要将毫米单位坐标转换为板卡编码值，并根据 Mark、Jump、Begin、End 等状态写入不同操作码。若启用 Y/Z 轴互换，执行层还会在编码前交换逻辑 Y、Z 与硬件通道的对应关系。

双缓冲发送算法用于提高 TCP 数据下发的连续性。系统在一侧缓冲区填充新生成的数据帧，同时由发送线程处理另一侧缓冲区中的待发送数据。当缓冲区满或任务结束时，通过强制入队将数据提交给发送线程。该算法将帧生成和网络发送解耦，减少通信等待对主界面的影响。需要注意的是，缓冲排空只代表发送侧任务完成，并不等同于振镜物理运动完成，因此界面进度和设备运动反馈仍应在逻辑上区分。

### 2.6.10 本节小结

综上，NBCAM 软件涉及的算法可以分为几何处理类、映射类、路径规划类和执行编码类。曲面网格预处理、Patch 聚类和参数化解决三维曲面如何展开的问题；SVG 解析、纹理映射和路径生成解决二维图案如何转化为加工轨迹的问题；UV 反映射、路径段组织和工艺参数绑定解决轨迹如何成为加工任务的问题；数据帧编码和双缓冲发送则解决加工任务如何进入硬件的问题。这些算法共同构成了本文系统从复杂曲面模型到激光加工数据帧的完整技术链路。

## 2.7 本章小结

本章介绍了 NBCAM 软件系统的需求分析、总体架构、图形界面、三维显示、核心数据模型以及主要功能的目的、原理与实现方式。系统采用分层结构将图形交互、算法控制、核心几何算法和执行通信分离，为后续纹理映射算法、曲面路径生成和硬件联调提供了工程基础，也为论文后续章节中对具体算法和通信协议的展开描述建立了统一语境。

# 3 基于纹理映射的复杂曲面图案映射方法

## 3.1 曲面 Patch 聚类与局部参数化

在本项目中，曲面预处理是模型导入之后、Patch 聚类和参数化之前的基础步骤，其目的在于把外部三维模型整理为适合后续算法处理和设备加工的统一几何数据。复杂曲面模型通常来源于 CAD、逆向扫描或第三方建模软件，不同来源的模型在单位尺度、坐标原点、法向方向、三角面质量和拓扑连续性上可能存在明显差异。如果直接在原始模型上执行纹理映射和路径规划，容易出现 Patch 划分不稳定、UV 展开失败、路径越界或加工坐标超出软限位等问题。因此，曲面预处理并不是可有可无的显示步骤，而是保证后续纹理映射、路径生成和数据帧下发可靠性的前置环节。

曲面预处理首先需要完成模型数据的标准化。系统在读取 STL/OBJ 模型后，将顶点、三角面片、法向量和面片索引转换为内部统一的 `TriangleMesh` 数据结构，并计算模型包围盒、中心点和尺度范围。包围盒用于判断模型是否落在加工区域内，中心点用于自动居中和偏置计算，尺度范围则用于检查模型单位是否符合毫米级加工需求。对于实际加工平台而言，坐标值最终要编码到板卡数据帧中，若模型尺寸过大或原点偏移过远，即使几何形状正确，也可能在执行阶段触发软限位保护。因此，系统提供模型偏置、自动居中和加工区域显示功能，使用户能够在路径规划之前确认模型与设备工作空间之间的关系。

其次，曲面预处理需要关注三角网格质量。参数化算法和 UV 到 XYZ 的反映射都依赖三角面片的局部几何关系，若模型中存在退化三角形、重复点、孤立面片或严重不均匀的长细三角形，就可能导致局部面积接近零、重心坐标计算不稳定或 UV 三角形翻转。项目中的预处理流程会尽量保持原始模型形状不变，同时通过网格索引整理、法向检查、邻接关系构建和必要的细分操作，为后续 Patch 聚类提供稳定输入。对于局部面片过大、路径落点过稀的模型，系统保留自适应均匀细分功能，使曲面在几何表达上更加细致，从而提高纹理贴附和曲面路径显示的连续性。

再次，曲面预处理还承担曲面语义准备的作用。复杂曲面并不是一个整体平面，后续加工通常只发生在某个局部区域，因此系统需要在预处理阶段建立面片邻接关系、法向变化关系和局部几何连续性信息。这些信息会被 Patch 聚类模块使用，用于判断哪些三角面片可以归入同一加工区域。若没有预先建立可靠的邻接结构，聚类算法只能把面片视为孤立元素，难以识别曲面边界、折线和自然分区。通过预处理得到的法向、二面角和拓扑邻接信息，系统可以把几何连续的区域组织成 Patch，为局部参数化提供边界明确、形状相对平滑的输入。

最后，曲面预处理还服务于软件可视化和人机交互。系统会把处理后的网格转换为 VTK 可渲染数据，在三维视图中显示模型、坐标轴、加工区域和 Patch 高亮结果。用户可以通过视图判断模型是否倒置、是否偏离平台中心、是否存在明显破面或异常尺度。这个过程看似属于界面显示，实际上是加工安全检查的一部分。只有当模型几何、坐标范围和显示状态都确认无误后，才适合继续执行参数化、SVG 纹理贴附和路径规划。因此，本文将曲面预处理视为从“外部模型文件”到“可加工曲面对象”的转换过程，它在数据结构、几何质量、坐标约束和交互验证之间建立了稳定桥梁。

复杂曲面通常由大量三角面片构成，直接对整个模型参数化可能导致变形较大或 UV 区域难以控制。本文系统首先对模型进行 Patch 聚类，将模型划分为若干局部曲面区域。用户可以在 3D 视图中选择需要加工的 Patch，并对该 Patch 执行局部参数化。

Patch 聚类主要依据相邻三角面片法向夹角和二面角阈值，将法向变化较小、几何连续性较好的面片聚为一组。该方法适合将复杂模型划分为若干加工区域，便于后续纹理映射和路径规划。

局部参数化将三维 Patch 映射到二维 UV 平面。系统已接入 LSCM、ARAP 和 Authalic 等参数化算法。LSCM 倾向于保持局部角度，适合图案形状保持；ARAP 倾向于保持局部刚性，适合降低局部长度变形；Authalic 倾向于保持面积关系。不同参数化方法适用于不同曲面形状和加工需求。

从功能目的看，Patch 聚类不是单纯为了显示彩色分区，而是为了把后续所有二维图案操作限制在一个局部可控的曲面区域内。复杂曲面上不同区域的法向、曲率和边界条件差异较大，若强行将整张模型展开到一个 UV 平面，参数化结果往往会出现严重拉伸，甚至可能导致纹理区域交叠。通过先聚类再局部展开，系统把复杂全局问题拆解为多个局部问题，使参数化算法、纹理编辑和路径生成都拥有明确作用域。

从工程实现看，系统需要同时维护全局模型索引和 Patch 内局部索引。全局模型索引用于在 3D 视图中显示完整模型和响应用户选区，Patch 内局部索引用于参数化算法构造局部网格。二者之间的映射关系十分重要，因为 UV 坐标虽然在局部 Patch 中生成，但路径最终仍需回到全局三维坐标系中。若索引映射不稳定，就会出现 UV 视图显示正确而三维路径落到错误面片上的问题。

参数化结果的质量通常可从三个方面判断。第一是边界是否合理，若边界过于扭曲，后续 SVG 图案会在边缘处变形明显；第二是三角形是否翻转，若 UV 三角形方向错误，则重心插值会出现错误映射；第三是面积和长度畸变是否在可接受范围内。本文系统在界面上提供 UV 视图，就是为了让用户在路径规划前先观察这些问题。换言之，UV 视图不是辅助装饰，而是参数化功能的质量检查窗口。

参数化结果为每个模型顶点生成 UV 坐标：

```text
P_i = (x_i, y_i, z_i)  ->  Q_i = (u_i, v_i)
```

其中 `P_i` 为三维曲面顶点，`Q_i` 为二维参数域坐标。后续纹理贴附和路径生成都在该 UV 域中进行。

局部参数化还承担着“统一图案尺度”的作用。二维 SVG 图案本身使用文件坐标或像素坐标，三维模型使用毫米或模型单位，二者没有天然对应关系。UV 域提供了一个中间坐标系统，使软件能够先在二维域中完成图案布局和路径生成，再通过参数化映射回三维空间。这样一来，图案变换、路径填充和曲面映射之间形成稳定顺序：先确定局部曲面区域，再确定二维域中的图案位置，最后再回到真实加工坐标。

## 3.2 SVG 图案解析与纹理贴附

系统使用 nanosvg 解析 SVG 文件，获取图案轮廓、填充形状和画布边界。SVG 图案一方面用于可视化纹理显示，另一方面用于生成加工路径区域。

引入 SVG 的目的在于让加工图案保持矢量语义。与普通位图不同，SVG 保存的是直线、曲线、轮廓、填充和变换等几何信息，因此在缩放时不会产生明显锯齿，也更适合转化为加工路径。对于复杂曲面加工，图案文件往往来自平面设计软件，如果软件只能接收离散点，则会丢失大量原始轮廓信息；而使用 SVG 可以在较高层次上理解图案结构，为轮廓加工、区域填充和纹理预览提供共同输入。

在可视化贴图中，系统首先将 SVG 栅格化为指定分辨率图像，然后创建 VTK 纹理对象。对于选中 Patch 的每个顶点，系统根据其 UV 坐标计算对应纹理坐标：

```text
(u, v) -> (s, t)
```

其中 `(u, v)` 为 Patch 参数化坐标，`(s, t)` 为归一化纹理坐标。由于 SVG 图像坐标系和 VTK 纹理坐标系方向不同，系统在生成纹理坐标时对 V 方向进行翻转。贴图过程本质上是将二维 SVG 图案通过 UV 坐标附着到三维曲面 Patch 上。

该过程在软件中表现为：用户选择 Patch 后执行“应用 SVG 纹理到 Patch”，SVG 图案即可显示在三维曲面表面和 UV 视图中。此时纹理不仅是显示对象，也会被保存到 `TextureInfo` 中，作为后续路径规划的输入。

这里需要区分“纹理显示”和“路径区域”两个概念。纹理显示主要服务于用户观察，它把 SVG 渲染为图像并贴到三维 Patch 上；路径区域则服务于加工，它需要知道哪些位置属于图案内部、哪些位置属于外部、轮廓在哪里、填充线应从哪里进入和退出。二者使用同一个 SVG 来源和同一套 UV 变换参数，但最终进入不同模块：显示模块关注视觉效果，路径模块关注几何可执行性。

为了保证显示和加工一致，系统将纹理文件路径、分辨率、透明度、材料参数以及几何变换参数一并保存。这样做的目的，是让后续重新规划路径时不需要用户重复调整纹理位置，也避免出现三维视图中看到的是一种图案位置，而路径生成使用另一种旧参数的情况。在 CAM 软件中，可重复性非常重要，同一模型、同一 Patch、同一 SVG 和同一组纹理参数，应当能够生成一致的路径结果。

`TextureInfo` 记录的主要信息包括：

```text
patch_id
svg_filepath
texture_width, texture_height
opacity, ambient, diffuse, specular
scale_x, scale_y
translate_x, translate_y
rotation_deg
enable_inverse_stretch_prewarp
inverse_stretch_prewarp_strength
```

## 3.3 纹理变换与逆向预扭曲补偿

为了使 SVG 图案能够准确放置在目标曲面区域，系统提供纹理编辑功能。用户可以对纹理执行缩放、平移和旋转。纹理变换以 Patch UV 范围中心为参考点，基本过程如下：

```text
q = (u, v)
q_c = q - c
q_s = S q_c
q_r = R(theta) q_s
q_t = q_r + t + c
```

其中 `c` 为 UV 区域中心，`S` 为缩放矩阵，`R(theta)` 为旋转矩阵，`t` 为平移向量。变换后的 UV 坐标用于纹理显示和路径区域映射。

纹理变换功能的目的，是解决二维图案与曲面加工区域之间的人工配准问题。实际应用中，SVG 图案的设计坐标原点、宽高比例和方向不一定与模型 Patch 的 UV 范围匹配。如果没有缩放、平移和旋转功能，用户只能回到外部软件中反复修改 SVG 文件，这既低效也难以保证精确。将这些变换放入 CAM 系统内部，可以让用户在三维曲面上直接观察调整结果。

从数学上看，缩放控制图案在 UV 域中的占比，平移控制图案中心位置，旋转控制图案主方向。系统以 Patch UV 范围中心或纹理自身中心作为参考点，是为了避免缩放和旋转时图案发生不期望的大范围漂移。若以坐标原点作为旋转中心，局部小角度调整也可能导致图案远离目标区域，不符合交互直觉。

由于曲面参数化会引入局部拉伸，UV 域中等间距或等比例的图案映射回三维曲面后可能发生变形。为此，系统提供逆向预扭曲补偿接口。该方法通过估计 Patch 上 UV 到三维曲面的局部拉伸情况，在生成图案轮廓和路径前对 UV 区域做反向补偿，使映射到三维曲面后的图案更接近目标形状。

当前系统中的逆向预扭曲补偿主要作为工程接口和可调参数保留，用户可以启用或关闭补偿，并设置补偿强度。后续可进一步结合实际加工测量结果，对补偿模型进行标定。

逆向预扭曲补偿的意义在于承认一个事实：参数化不可能在一般复杂曲面上同时完全保持角度、面积和长度。若直接把平面图案映射到曲面，图案在高曲率区域会发生拉伸或压缩。预扭曲的思想是先在 UV 域中对图案做一个反方向变形，使其经过曲面映射后更接近目标形状。例如某区域从 UV 到三维后沿某方向被拉长，则预处理时可适当压缩该方向。当前系统保留补偿强度参数，便于后续根据校正样件或显微测量结果建立更精确的补偿模型。

## 3.4 基于纹理区域的 UV 路径生成

在路径规划阶段，系统读取当前 Patch 的 `TextureInfo`，将 SVG 图案边界映射到 Patch UV 范围内，并以该纹理区域作为加工区域。与仅显示纹理不同，该过程会直接影响加工路径生成。

系统支持多种路径策略：

1. 轴向往返填充：在图案区域内生成往返扫描线，适合一般填充加工。
2. 圆弧往返填充：围绕指定圆心和半径生成圆弧型扫描路径，适合特定曲面结构。
3. 回型填充：由边界向内或由内向外生成回型路径。
4. 轮廓加工：沿 SVG 边界生成轮廓加工路径。
5. Z 轴分层填充：按层高生成分层路径，适合具有高度层次的加工需求。

多种路径策略的存在，是因为不同图案和工艺目标对路径形态有不同要求。轴向往返填充路径生成简单、覆盖均匀，适合大多数实心区域；圆弧往返填充适合具有圆形或环形趋势的图案，可以减少路径与边界夹角突变；回型填充更接近轮廓内缩过程，适合希望保留外形连续性的区域；轮廓加工用于只加工图案边界或先描边后填充的场景；Z 轴分层填充则为具有高度或多层加工需求的任务提供接口。

从算法角度看，填充策略本质上是在 UV 域中构造一族采样线或采样曲线，并求这些曲线与 SVG 区域的交集。以轴向往返填充为例，系统按照填充角度生成一组等间距扫描线，再计算扫描线落入图案区域内的有效线段。为了提高加工效率，相邻有效线段之间通常采用往返连接方式，减少长距离空跳。对于回型填充，则需要对边界做偏置或逐层收缩，直到内部区域无法继续生成有效轮廓。不同策略虽然路径形态不同，但最终都会转换为统一的 UV 路径点序列。

路径生成还需要处理边界、孔洞和断点。SVG 图案可能包含多个互不相连的区域，也可能包含内部孔洞。若填充线穿过孔洞，系统需要将其切分为多个有效段，并在段间插入 Jump 标记。`jump_before` 的作用就是把这些几何断裂传递给后续路径规划模块，使真实执行时激光能够在无效区域关闭，避免错误加工。

路径生成结果首先表现为 UV 路径点序列：

```text
UVPathPoint = (u, v, jump_before, is_arrow_tip)
```

其中 `jump_before` 用于标记路径断开，后续规划会根据该标记插入 Jump 段。生成路径后，系统在 UV 视图中显示路径，并在 3D 视图中通过反映射结果显示曲面路径。

基于纹理区域生成路径的优势在于：二维图案、曲面参数化和加工路径使用同一套 UV 坐标关系，能够保证显示纹理和实际加工区域的一致性。用户看到的 SVG 贴附位置，就是路径生成的参考区域。

在本文系统中，UV 路径生成阶段暂不直接关心板卡协议、TCP 发送或振镜轴映射。这种分离是有意设计的：路径生成模块只负责回答“曲面上哪里需要加工”，执行模块再负责回答“怎样把这些点发给设备”。将二者分开，可以避免硬件协议变化影响算法层，也便于后续加入 RTC-6、五轴平台或其他控制器。

## 3.5 本章小结

本章介绍了本文系统中的纹理映射方法。系统首先对选定曲面 Patch 进行局部参数化，然后将 SVG 图案作为纹理映射到 Patch UV 域，并通过纹理变换和逆向预扭曲补偿接口调整图案位置与变形。最后，系统基于纹理覆盖区域生成 UV 加工路径。该方法使二维图案能够自然地参与复杂曲面加工路径生成，是本文系统的核心算法链路之一。

# 4 曲面加工路径生成与数据帧下发

## 4.1 UV 路径到三维路径的映射

UV 路径生成后，需要将其转换为三维曲面上的加工路径。系统通过 `UVMapper` 实现该过程。对于每个 UV 路径点 `(u, v)`，系统首先在参数化后的三角网格中查找包含该点的 UV 三角形，然后使用重心坐标插值计算其三维位置。

设 UV 平面上某三角形三个顶点为：

```text
Q_1 = (u_1, v_1), Q_2 = (u_2, v_2), Q_3 = (u_3, v_3)
```

其对应三维顶点为：

```text
P_1 = (x_1, y_1, z_1), P_2 = (x_2, y_2, z_2), P_3 = (x_3, y_3, z_3)
```

对于 UV 点 `Q`，若其重心坐标为：

```text
Q = alpha Q_1 + beta Q_2 + gamma Q_3
alpha + beta + gamma = 1
```

则其三维点为：

```text
P = alpha P_1 + beta P_2 + gamma P_3
```

为了提高大模型下的查找效率，系统在三角面数量较大时构建 UV 空间网格，将三角形索引分配到网格单元中，减少每个路径点的候选三角形数量。若网格查找失败，则回退到线性查找，以提高鲁棒性。

UV 到 XYZ 映射功能的目的，是把二维参数域中的“设计路径”恢复为三维曲面上的“真实路径”。在纹理映射链路中，UV 域承担了图案编辑和路径生成的中间角色，但设备最终不能执行 UV 坐标，必须得到以毫米为单位的三维空间坐标。因此该功能直接决定了曲面图案能否落在正确的物理位置上。

重心坐标插值的优点在于简单、稳定且与三角网格天然匹配。对于三角网格而言，任意位于某个 UV 三角形内部的点都可以用三个顶点的线性组合表示。由于这三个 UV 顶点与三个三维顶点一一对应，因此同一组权重也可以用于三维空间插值。这样做等价于假设每个三角面片内部的映射是线性的，符合三角网格分片线性表示的基本思想。

查找包含 UV 点的三角形是该步骤中的核心问题。若路径点数量较少，可以直接遍历全部 UV 三角形；但实际填充路径往往包含大量点，直接遍历会导致规划速度随模型规模明显下降。因此系统在三角面数量较多时引入 UV 空间网格，将三角形按其 UV 包围盒分配到网格单元中。路径点映射时只需检查所在单元附近的候选三角形，从而降低平均查找开销。若由于边界误差、浮点误差或网格划分导致加速查找失败，系统再回退到线性查找，优先保证结果可靠。

映射后的路径点保存为 `PathPoint`：

```text
PathPoint = (u, v, x, y, z, a, b, laser, params_override)
```

其中 `laser` 用于表示该点是否为有效加工点。如果 UV 点无法映射到任何三角形，系统会将其标记为无效点，并在后续过滤阶段形成路径断点。

无法映射的点并不一定意味着整个任务失败。它可能出现在 SVG 区域略微超出 Patch UV 边界、边界点正好落在三角形边上、或者参数化局部存在退化三角形时。系统将这些点标记为无效并在后续形成断点，是为了最大限度保留有效路径，同时避免硬件执行超出模型区域的坐标。对于真实加工而言，宁可在边界处形成可见断开，也不能把无效点强行编码为错误坐标。

## 4.2 LaserJob 加工任务模型

为了统一路径预览、模拟加工和真实下发，系统将三维路径组织为 `LaserJob`。该模型是算法层和执行层之间的主要数据契约。

`LaserJob` 的主要组成如下：

```text
LaserJob
  meta
  coordinate
  parameterization
  process_defaults
  segments[]
```

其中 `segments` 是加工路径段集合。每个 `PathSegment` 包含段 ID、段类型、路径策略、段级工艺参数和点列表。段类型主要包括：

- `MARK`：激光加工段。
- `JUMP`：激光关闭跳转段。

路径规划模块会根据 UV 路径断点、点间关系和激光状态生成 Mark 段与 Jump 段。真实执行时，Mark 段会根据右侧“激光输出/开光”选项决定是否输出开光位；Jump 段始终关光。

`LaserJob` 模型的优势在于将几何路径、工艺参数和执行属性统一封装，使系统能够在不同执行后端之间复用同一任务数据。例如默认板卡后端和 RTC-6 后端均可基于 `LaserJob` 执行预检和编码。

`LaserJob` 的设计目的还包括持久化和可追溯。一个加工任务不仅包含点，还包含源模型、参数化方法、工艺参数、坐标变换和路径段。将这些信息一起保存，有利于后续复现实验，也有利于论文实验部分记录“同一任务在不同参数下的加工结果”。如果只保存最终数据帧，虽然可以再次发送，但无法追溯路径如何生成；如果只保存 SVG 和模型，又无法复现当时的路径规划参数。因此 `LaserJob` 位于两者之间，既接近算法结果，又保留执行所需语义。

段级组织是 `LaserJob` 中最重要的结构之一。激光加工路径不是一条连续曲线，而是由加工段和空跳段交替组成。Mark 段代表激光打开并沿目标路径运动，Jump 段代表激光关闭并移动到下一加工起点。将二者明确区分，可以在预览中显示路径结构，在执行中控制开关光，在日志中判断某一帧对应加工还是移动。特别是在多个 SVG 子区域或多个 Patch 加工时，段级组织能显著降低调试难度。

## 4.3 路径预览与模拟加工

路径预览窗口用于检查加工路径的段级信息。表格中显示段 ID、路径类型、激光状态、起点 XYZ、终点 XYZ、起点 UV、终点 UV、点数和长度。用户可以筛选路径段、聚焦某一段、导出 CSV 或保存当前 Patch 路径。

模拟加工用于在不下发硬件数据的情况下检查路径顺序和显示逻辑。模拟过程中：

1. 路径预览表格只选中当前正在加工的段。
2. 3D 视图累计高亮已加工段。
3. UV 视图累计绘制已加工路径。
4. 状态栏显示当前段进度。

模拟加工的作用不是替代执行器预检，而是在 UI 层帮助用户确认路径顺序、路径区域和图案位置。真实出光前，建议先完成模拟加工，再进行不开光轨迹试跑和校正图案验证。

路径预览还承担“用户确认”的功能。复杂曲面路径通常点数较多，若只在三维视图中显示整条线，用户很难发现某一段的起点、终点或类型是否异常。表格化预览可以让用户按段筛选，查看某一段的长度、点数和坐标范围。例如某段长度异常大，通常说明存在不合理空跳；某段点数过少，可能说明填充线与图案区域交集过短；某段起终点跨越模型边界，则可能说明参数化或纹理范围存在问题。

模拟加工与真实加工之间必须保持清晰边界。模拟加工只验证软件路径顺序和界面高亮，它不代表 TCP 已发送、不代表板卡已接收，也不代表振镜已经运动。因此在系统设计中，模拟加工不应被用作硬件完成信号，而应被视为真实加工前的逻辑预演。这样的分层可以避免误把 UI 动画结果当成设备反馈。

## 4.4 默认板卡 TCP 数据帧生成

默认板卡执行链路采用固定 16 字节帧格式，每帧包含 8 个小端 `uint16_t` 字段：

```text
B, A, Z, Y, X, OPCODE, arg7, arg8
```

常用帧类型如下：

| 帧类型 | OPCODE | 说明 |
|---|---:|---|
| Begin | `0xFF00` | 加工开始标记 |
| Mark | `0x00FF` | 激光打开的加工点 |
| Jump | `0x0000` | 激光关闭的运动点 |
| End | `0x1100` | 加工结束标记 |
| Frequency | `0xAA00/0x5500` | 频率设置相关帧 |
| Power | `0xBB00` | 功率设置相关帧 |

坐标编码采用如下形式：

```text
encoded = round(mm * 1000 + 32768)
```

其中 `mm` 为经过模型变换、轴偏置和 Y/Z 轴互换后的机床坐标。编码前系统会检查软限位，防止超范围坐标进入数据帧。

数据帧字段顺序采用 `B,A,Z,Y,X,OPCODE,arg7,arg8`，说明执行器层必须显式处理逻辑坐标与板卡协议字段之间的差异。对于上层路径规划而言，点的自然顺序通常是 `X,Y,Z,A,B`；但板卡协议按自身硬件通道顺序组织字段。若在多个地方随意转换，很容易出现某一功能正常、另一功能轴向错误的情况。因此系统将轴映射和编码集中在执行层完成，并通过 frame 日志记录最终写入帧的坐标值。

开关光控制是数据帧生成中的另一个关键点。系统右侧“激光输出/开光”选项决定 Mark 段是否真正写入开光操作码。未勾选时，即使路径段逻辑上属于 Mark，发送帧也应按关光轨迹试跑处理；勾选时，Mark 段才写入开光帧。这样设计的目的，是让用户能够在不出光的情况下验证运动轨迹、轴向和范围，确认无误后再进行真实加工。Jump 段无论是否勾选开光，都应保持关光，以避免段间移动留下多余加工痕迹。

系统当前支持 `swap_yz_axes` 轴互换开关。该开关用于处理模型逻辑坐标与振镜实际通道不一致的问题。勾选后：

```text
frame_y = logic_z
frame_z = logic_y
```

该映射会同时影响真实加工、手动操作、校正图案和 frame 日志，保证显示、日志和发送帧的一致性。

延时帧用于适配激光和振镜的物理响应。实际设备中，激光开光、关光、振镜跳转、扫描拐角和多边形连接都可能存在响应滞后。若完全按几何点序列发送，可能出现起点烧蚀不足、终点过烧、拐角圆滑或轮廓闭合不良。系统中默认的开光延时、关光延时、mark 延时、jump 延时和 polygon 延时，目的就是在关键状态变化处插入保持帧，让硬件有足够时间完成状态切换。需要注意的是，延时参数应根据实际板卡节拍和样件效果调试，论文中不应将默认值视为所有设备通用的最佳值。

数据下发采用 `DataBuffer` 双缓冲机制。加工批次通过 `appendBytes()` 写入缓冲区，缓冲区满后交由 TCP 线程发送。加工结束时，系统使用 End 帧和零填充补齐尾部，并调用 `forceFill()` 强制入队。`waitUntilIdle()` 用于等待发送侧队列排空，但它只代表传输侧完成，并不等价于振镜物理运动完成。

采用双缓冲的主要原因，是加工路径数据往往远大于单次网络写入或板卡缓存。若主线程一边生成帧一边阻塞等待 TCP 发送完成，界面会失去响应，路径高亮也难以连续更新。双缓冲把数据生成和网络发送拆开：生成端只负责持续填充缓冲区，发送端根据板卡请求或连接状态取出缓冲区发送。这样可以在一定程度上平滑数据流，减少由于网络等待造成的主循环停顿。

`forceFill()` 的作用是把未填满的尾部缓冲也提交给发送线程。由于板卡协议或旧有数据生成逻辑可能以固定块为单位接收，尾部数据不足一个完整块时需要补齐零值或空帧，避免最后一段路径丢失。这里的零填充不是路径算法的一部分，而是传输对齐的一部分。论文中需要明确这一点：填充零并不表示要让振镜执行大量零坐标运动，正确实现应保证有效载荷长度、帧类型和尾部补齐语义清晰可查。

`waitUntilIdle()` 常被误解为“加工完成”。实际上它只能说明发送侧缓冲区和队列达到空闲状态，即软件层认为数据已经提交完成。真实振镜是否运动完毕，取决于板卡内部缓存、执行节拍和硬件反馈协议。若没有可靠的设备完成信号，就不能把界面进度直接绑定到发送排空。本文系统在设计说明中将二者分离，是为了避免界面进度、TCP 排空和物理运动三者互相混淆。

## 4.5 本章小结

本章介绍了曲面加工路径从 UV 到 XYZ 的映射方法、LaserJob 任务模型、路径预览与模拟加工机制，以及默认板卡 TCP 数据帧生成和下发方式。通过统一任务模型和固定帧协议，系统将纹理映射生成的曲面路径转换为硬件可执行的数据流，实现了从算法路径到设备控制的连接。

在软件层面，验证重点包括：

2. Patch 是否能够聚类和选择。

## 5.2 通信链路验证

通信链路验证主要面向默认板卡 TCP 发送路径。系统提供连接设置、TCP 测试、串口测试和执行前联锁检查。默认 TCP 参数包括主机 IP、端口、超时、重试间隔和最大重试次数。

当前 TCP 数据发送采用板卡触发式思路，即 TCP 线程在接收板卡反馈或请求后发送缓冲区数据。系统在发送过程中记录最近一次传输反馈，并在排空超时时输出缓冲区状态，包括线程运行状态、连接状态、读队列、写队列、在途缓冲和当前写指针。

通信链路验证之所以重要，是因为这部分直接决定“软件里生成的东西”能否被硬件真正接收。复杂曲面加工系统中，几何算法可能运行正常，但如果帧长度、字节序、发送节奏或开关光位错误，最终设备输出就会完全偏离预期。尤其是 TCP 方式下，发送端和接收端通常还存在握手、请求或反馈机制，因此不能把“socket 已连接”简单等同于“加工数据已被板卡执行”。

当前实现将发送过程中的反馈和排空状态一起记录，目的是让开发者观察发送链路是否存在阻塞、重试或等待过久的现象。若在日志中看到发送缓冲长期不空，通常说明对端没有及时请求、网络阻塞或帧块组织不合理；若发送一切正常但设备运动异常，则问题更可能出在帧内容、坐标映射或设备侧节拍。将这些层次区分开，有助于缩小故障范围。

通信验证建议按以下顺序进行：

1. 仅测试 TCP 连接，不发送加工任务。
2. 生成 frame 日志，核对首帧、Mark 帧、Jump 帧和 End 帧格式。
3. 取消右侧开光选项，执行小范围轨迹试跑。
4. 检查振镜方向、零点位置和软限位。
5. 勾选开光后再进行真实样件加工。

## 5.3 手动点动与 Y/Z 轴互换验证

实际设备联调中，模型坐标轴和振镜通道可能不一致。为此，系统在“设备设置”中提供“交换 y/z 轴”选项，并在手动操作界面提供 X/Y/Z 三轴点动。

手动操作界面包含步长输入、各轴正负方向按钮和 STOP 回中按钮。点击方向按钮后，系统会根据当前逻辑坐标和步长计算目标位置，检查软限位，再生成 TCP 数据帧发送给板卡。STOP 回中会将 X/Y/Z 位置移动到各自软限位中心。

手动点动的验证目标为：

1. X 轴正负方向是否与台面预期一致。
2. Y 轴和 Z 轴是否需要互换。
3. 目标位置是否受到软限位保护。
4. spdlog 中逻辑坐标、frame 轴坐标和编码值是否一致。

手动点动之所以要单独验证，是因为它最适合快速暴露轴映射问题。自动加工路径往往是连续且复杂的，路径错误可能在运行很久后才显现；而手动点动只发送一个方向和固定步长，可以立即看出坐标正负是否与预期一致。对于本系统来说，这一步尤其重要，因为模型坐标、加工平面定义和振镜真实通道之间可能存在轴顺序差异，因此系统增加了“交换 y/z 轴”选项，并要求日志中同步记录交换后的轴值，便于在定位问题时直接观察编码前后的对应关系。

STOP 回中按钮不仅是“停下来”，更是一种状态复位。对于长时间的联调来说，设备可能已经偏离零点或接近软限位，此时继续点动容易触发保护或引入额外偏差。将设备移动到软限位中点，可以在不依赖外部回零装置的情况下，把控制目标恢复到一个已知中间状态，便于继续验证其他方向。

若发现振镜运动方向与模型路径不一致，应优先检查 `swap_yz_axes` 是否正确设置，而不是修改路径生成算法。该设计将硬件通道映射集中到执行编码阶段，使纹理映射、路径生成和显示逻辑保持稳定。


默认校正图案为 3x3 正方形阵列：

- 正方形边长：2 mm。
- 相邻中心间距：5 mm。
- 中心正方形中心：`X=0, Y=0, Z=0`。
- 九个正方形中心：`X/Y = -5, 0, 5 mm`。

系统生成校正图案时，以逻辑 XY 平面为加工平面，Z 固定为 0。若启用 Y/Z 轴互换，则在数据帧编码阶段将逻辑 Y 写入 frame Z 通道，将逻辑 Z 写入 frame Y 通道。每条正方形边都按当前速度进行插补，并使用当前开光延迟、关光延迟、扫描延迟、跳转延迟和多边形延迟生成保持帧。

校正图案验证流程如下：

1. 确认板卡 TCP 参数正确。
2. 确认软限位覆盖 `X/Y = -6 mm` 到 `6 mm`。
3. 首次发送时取消开光，仅观察轨迹方向。
4. 若方向错误，切换“交换 y/z 轴”后重新验证。
5. 方向和中心正确后勾选开光，打标 3x3 正方形。
6. 测量正方形边长、中心间距和整体位置偏差。

校正图案的设计原则是“简单但信息密度高”。单个正方形可以检查局部边长是否正确，3x3 阵列可以同时观察中心点、正负方向、四角对称性和全局尺度。若中心正方形偏离原点，通常说明零点或坐标偏置有问题；若左右、前后间距不一致，说明轴比例或轴互换存在误差；若上边和下边长度不同，则说明扫描方向或延时参数可能影响了局部尺寸。与复杂标识图案相比，正方形阵列更容易通过肉眼和卡尺测量形成判断，因此适合作为初始校正方案。

在本文系统中，3x3 校正图案还承担一个额外作用，即验证“开光”和“不开光”两种状态下发送逻辑是否一致。不出光时，操作者可以先检查路径是否位于正确平面、是否超出软限位、是否存在方向翻转；出光后，再进一步检查样件实际尺寸是否与设计值一致。这样做能够把设备安全性和加工精度验证分开，降低试错成本。

如果系统启用了 Y/Z 轴互换，校正图案的坐标解释也要同步变化。也就是说，视觉上定义的前后左右，与发送到硬件时的轴通道对应关系必须保持一致，否则校正结果会和真实加工结果不一致。该要求再次说明，校正不是一个孤立功能，而是整个坐标链路正确性的综合检验。

# 6 总结与展望

## 6.1 本文主要内容及结论

本文围绕复杂曲面激光加工中的二维图案映射和加工路径生成问题，研究并实现了一套基于纹理映射的复杂曲面激光加工 CAM 软件系统。本文主要工作和结论如下。

1. 设计并实现了 NBCAM 软件系统总体架构。系统采用 Qt 图形界面、ApplicationController 控制层、core 核心算法库和 executor 执行通信层的分层结构，实现了模型处理、纹理映射、路径生成、路径预览和数据帧下发的完整流程。

2. 实现了曲面 Patch 聚类与局部参数化功能。系统能够对导入的 STL/OBJ 模型进行显示、聚类、Patch 选择和局部参数化，并在 UV 视图中显示参数化结果，为纹理映射提供二维参数域。

3. 提出了并实现了基于纹理映射的曲面图案加工方法。系统将 SVG 图案作为纹理贴附到选定 Patch 的 UV 域，并将纹理区域作为加工路径生成依据，实现了“所见纹理区域”和“实际加工区域”的统一。

4. 实现了 UV 路径到三维曲面路径的反映射。系统通过 UV 三角形查找和重心坐标插值，将 UV 加工路径转换为三维 `PathPoint`，并组织为统一的 `LaserJob` 任务模型。

5. 实现了默认板卡 TCP 数据帧下发链路。系统支持固定 16 字节帧格式、DataBuffer 双缓冲、TCP 发送线程、开关光控制、Y/Z 轴互换、frame 日志导出和发送状态反馈，为真实硬件联调提供了基础。

6. 实现了面向联调的手动操作和加工平面校正功能。通过手动点动和 3x3 正方形校正图案，系统能够在复杂路径真实加工前验证坐标方向、中心位置、比例和开光状态。

综上，本文系统已打通从复杂曲面模型到曲面纹理映射路径，再到板卡数据帧下发的主要技术链路，为复杂曲面激光加工软件系统的进一步实验验证和工程应用提供了基础。

## 6.2 本文的主要创新点

本文的主要创新点和工程特色如下。

1. 将计算机图形学中的纹理映射思想用于复杂曲面激光加工路径生成。系统中的 SVG 纹理不仅用于显示，还作为加工区域和路径生成的约束，实现了图案贴附与路径规划的一致性。

2. 构建了基于 Patch 参数化的复杂曲面加工流程。通过局部参数化降低全局曲面展开难度，使复杂曲面局部区域能够使用二维图案和二维填充策略生成加工路径。

3. 建立了统一的 LaserJob 加工任务模型。该模型同时服务于路径预览、模拟加工、日志导出和执行下发，提高了系统各阶段数据的一致性。

4. 实现了面向实际硬件联调的数据帧发送机制。系统将 Y/Z 轴互换、开关光控制、双缓冲发送、手动点动和校正图案纳入同一执行链路，提高了软件与板卡联调的可观测性和可维护性。

5. 预留了曲面变参数加工和逆向预扭曲补偿接口。系统已具备将曲率、拉伸和工艺参数进一步引入路径生成与执行链路的扩展基础。

## 6.3 后续工作展望

虽然本文系统已实现主要软件流程，但仍有若干方面需要进一步完善。

1. 完善加工实验验证。当前系统已具备数据帧下发和校正图案功能，后续需要通过实际样件加工，测量图案尺寸、边界误差、槽宽、槽深和表面质量，并形成定量实验结果。

2. 建立振镜几何校正模型。3x3 正方形校正可以作为初始验证工具，后续可扩展为 9x9 或 11x11 网格，通过测量误差拟合振镜畸变补偿模型。

3. 优化纹理映射误差补偿。当前逆向预扭曲补偿仍为可调接口，后续应结合参数化局部拉伸和实测图案误差，建立更精确的补偿模型。

4. 完善曲面变参数加工。后续可根据曲率、入射角、光斑直径、脉冲重叠率和单位面积能量，动态调整功率和速度，提高复杂曲面加工均匀性。

5. 增强自动化测试和异常恢复。应补充核心算法单元测试、典型模型回归测试、通信异常测试和日志分析工具，提高系统稳定性。

6. 完成 RTC-6 与更多板卡后端联调。当前系统已保留 RTC-6 执行器接口，后续可进一步完善硬件调用、文件格式转换和多后端兼容策略。
