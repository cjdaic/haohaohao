# NBCAM 用户手册

版本基线：当前软件，2026-05-21  
适用对象：设备操作员、工艺工程师、设备联调人员  
主程序：`laser_cam_qt.exe`  
软件版本：`0.0.1`

## 1. 软件用途

NBCAM 用于复杂曲面激光加工/标刻。软件可以导入三维模型和 SVG 图案，将图案映射到模型 Patch 上，生成激光加工路径，并进行模拟加工或下发到板卡。

典型流程：

```text
打开模型
  -> 面片聚类
  -> 选择 Patch
  -> 参数化 Patch
  -> 应用 SVG 纹理
  -> 设置加工参数
  -> 执行路径规划
  -> 路径预览
  -> 模拟加工
  -> 连接/联锁检查
  -> 真实加工
```

真实加工前必须先完成模拟加工和硬件安全检查。

## 2. 启动软件

Release 版本：

```text
build/bin/Release/laser_cam_qt.exe
```

Debug 版本：

```text
build/bin/Debug/laser_cam_qt.exe
```

RelWithDebInfo 版本：

```text
build/bin/RelWithDebInfo/laser_cam_qt.exe
```

软件启动后默认最大化显示。主界面通常包含：

- 中间：3D 模型视图。
- UV 视图：显示 UV 坐标、SVG 纹理和 UV 路径。
- 右侧：参数面板、平面位姿、任务/路径相关按钮。图案参数区包含“激光输出/开光”选项。
- 右上角：开始、暂停、急停、刷新、模拟加工按钮。
- 顶部菜单：文件、视图、工具、连接、设备与板卡、手动操作、校正、测量、日志、语言、帮助。

如果启动失败并提示 Qt 平台插件问题，请确认可执行文件目录下存在：

```text
platforms/qwindows.dll
qt.conf
```

## 3. 基本加工流程

### 3.1 打开模型

菜单：

```text
文件 -> 打开模型
```

支持常用三维模型格式，例如 STL/OBJ。打开后模型会显示在 3D 视图中。

如果模型位置不合适，可使用：

```text
工具 -> 模型偏置与加工区域
```

该功能可显示加工区域黄框，并支持自动居中、手动偏置和重置变换。

### 3.2 面片聚类

菜单：

```text
工具 -> 面片聚类
```

软件会提示输入二面角阈值。阈值越小，Patch 划分越细；阈值越大，相邻曲面越容易合并。聚类完成后，可在 3D 视图中点击选择需要加工的 Patch。

### 3.3 参数化选中 Patch

先在 3D 视图中选中一个 Patch，然后执行：

```text
工具 -> 参数化选中Patch
```

可选算法包括：

- ARAP：当前对话框默认选项，适合保持局部形变。
- LSCM：常用于一般曲面展开。
- Authalic：适合需要面积保持倾向的场景。

参数化完成后，UV 视图会更新。若失败，通常需要重新选择 Patch、调整聚类阈值或检查模型质量。

### 3.4 应用 SVG 纹理

先选择并参数化 Patch，再执行：

```text
工具 -> 应用SVG纹理到Patch
```

选择 SVG 文件后，可设置纹理分辨率、透明度、环境光、漫反射、镜面参数等。应用后，SVG 会显示在 3D 视图和 UV 视图中。

纹理编辑器支持：

- 缩放。
- 平移。
- 旋转。
- 逆向预扭曲补偿。
- 重置。

### 3.5 设置加工参数

右侧参数面板包含全局加工参数和图案参数。

全局加工参数：

- 功率：W、kW、mW。
- 频率：Hz、kHz、MHz。
- 速度：mm/s、m/s、cm/s、in/s、ft/s。

图案参数：

- 填充间距：mm、cm、m、in、ft。
- 填充角度：度或弧度。
- 填充策略。
- 填充方向。
- 加工轮廓模式。
- 激光输出：勾选“开光”后，真实加工下发和 frame 日志按开光帧生成；未勾选时只下发运动轨迹。

填充策略：

- 轴向往返填充。
- 轴向往返填充(圆弧)。
- 回型填充(内->外)。
- 回型填充(外->内)。
- 轮廓偏置回型(忽略空洞)。
- Z轴分层填充。

填充方向：

- 按角度。
- 锁定 U 轴。
- 锁定 V 轴。

加工轮廓：

- 只加工轮廓。
- 不加工轮廓。
- 加工全部。

修改参数后，可点击：

```text
重新规划路径
```

### 3.6 执行路径规划

菜单：

```text
工具 -> 路径规划 -> 执行路径规划
```

路径规划成功后：

- 3D 视图显示加工路径。
- UV 视图显示 UV 路径。
- 路径预览表格可查看每个路径段。
- `文件 -> 执行任务` 和右上角开始加工按钮会变为可用。
- 日志目录中会按当前模型、SVG、Y/Z 轴互换和开光选项生成 frame 日志，便于联调。

若提示路径规划失败，请确认：

- 已加载模型。
- 已完成面片聚类。
- 已选择 Patch。
- 该 Patch 已参数化。
- 该 Patch 已应用 SVG 纹理。
- 填充间距大于 0，且不要小到生成过密路径。

## 4. 路径预览

点击主界面“路径预览”按钮打开路径预览窗口。

路径预览表格显示：

- 段 ID。
- 路径类型。
- 激光状态。
- 起点 XYZ。
- 终点 XYZ。
- 起点 UV。
- 终点 UV。
- 点数。
- 长度。

常用操作：

- 按 ID 范围筛选路径段。
- 按路径类型筛选。
- 点击表格段，3D/UV 视图同步高亮。
- 右键路径段，聚焦该段。
- 导出 CSV。
- 保存当前 Patch 路径。

说明：

- 模拟加工时，表格只选中当前正在加工的段。
- 3D/UV 视图会累计显示已加工段。
- 保存路径后，该 Patch 的路径状态会保留，除非执行移除路径、清理面片数据或重新生成相关路径。

## 5. 模拟加工

右上角按钮：

```text
模拟加工
```

模拟加工不会真实下发硬件数据，用于预检和观察路径执行顺序。真实加工前建议至少模拟一次完整路径。

模拟加工表现：

- 状态栏显示当前段进度。
- 路径预览表格只选中当前段。
- 3D 和 UV 视图中，已加工路径段会累计变色。
- 可使用暂停、急停、刷新按钮控制模拟进度。

如果模拟后半段变慢，通常是路径点或路径段过多导致显示压力增加。可尝试：

- 使用 Release 版本。
- 关闭 UV 视图。
- 增大填充间距。
- 先只规划小范围 Patch。

## 6. 设备与板卡设置

入口：

```text
设备与板卡
```

包含：

- 设备设置。
- 板卡设置。
- 激光器设置。

### 6.1 设备设置

菜单：

```text
设备与板卡 -> 设备设置
```

可选择设备：

- 默认板卡。
- RTC-6 板卡。

可选择轴模式：

- 三轴。
- 五轴。
- 交换 y/z 轴。

三轴模式：

- X/Y/Z 进入运动数据。
- A/B 轴使用默认中位逻辑。

五轴模式：

- X/Y/Z/A/B 都会参与默认板卡数据帧。
- A/B 用于振镜偏摆/旋转相关数据。

注意：RTC-6 当前主要使用 X/Y/Z 和 RTC 单位系数，A/B 轴模式主要面向默认板卡链路。

“交换 y/z 轴”用于处理模型坐标轴与振镜实际通道不一致的设备。勾选后，实际加工、手动操作、校正图案和日志中的 Y/Z 数据帧通道都会交换；请在真实出光前先用手动操作和校正图案确认方向。

### 6.2 板卡设置

菜单：

```text
设备与板卡 -> 板卡设置
```

可设置：

时序参数：

- 跳转速度。
- 开光延迟。
- 关光延迟。
- 扫描延迟。
- 跳转延迟。
- 多边形延迟。

RTC-6 参数：

- 卡号。
- 编码系数。
- 默认扫描速度。
- 默认跳转速度。
- 程序文件。

机床零点偏置：

- X/Y/Z/A/B 偏置。

软限位：

- X/Y/Z/A/B 最小值和最大值。

软件会检查软限位合法性，最小值不能大于最大值。真实下发前，坐标会在任务变换和零点偏置后再检查软限位。

默认时序参数为：

```text
开光延迟：100 us
关光延迟：580 us
扫描延迟：600 us
跳转延迟：150 us
多边形延迟：450 us
```

默认板卡常用范围为 `+-32.768 mm`，因为默认编码为：

```text
round(mm * 1000 + 32768)
```

### 6.3 激光器设置

菜单：

```text
设备与板卡 -> 激光器设置
```

默认激光器资料：

```text
reference/M8 100w 产品说明书(1).pdf
```

默认激光器：

- 型号：M8 100W。
- 额定功率：100 W。
- 功率范围：0 W 到 100 W。
- 频率范围：20000 Hz 到 60000 Hz。
- 控制方式：串口+TTL。

可编辑参数：

- 型号。
- 控制方式。
- 资料来源。
- 额定功率。
- 最小功率。
- 最大功率。
- 最小频率。
- 最大频率。
- 备注。
- 串口号。
- 波特率。
- 是否执行前测试激光器串口。

支持导入参数文件：

- JSON。
- CSV。
- TXT。
- INI。
- PDF。

说明：

- 文本类参数文件会尝试自动读取型号、功率、频率、串口等字段。
- PDF 文件不会自动解析具体参数，只会作为资料来源记录，需要人工确认字段。

## 7. 连接设置与通信测试

### 7.1 连接设置

菜单：

```text
连接 -> 连接
```

可配置：

- gRPC 端点。
- 是否连接时测试 gRPC。
- TCP 主机。
- TCP 端口。
- TCP 超时。
- TCP 重试间隔。
- 串口号。
- 串口波特率。
- 是否执行前联锁时同时测试串口。
- 是否执行前将任务坐标转换到机床坐标。
- 是否真实下发前强制做 TCP/串口联锁检查。

注意：Dry Run 选项已从连接设置中移除。预检和模拟请使用右上角“模拟加工”。

### 7.2 通信测试

菜单：

```text
连接 -> 测试gRPC通信
连接 -> 测试TCP通信
连接 -> 测试串口通信
```

真实加工前建议依次检查：

1. TCP 主机和端口是否可连通。
2. 串口号和波特率是否正确。
3. 激光器联锁是否启用并可通过。
4. gRPC 是否为当前联调所需链路。

TCP 默认参数通常为：

```text
主机：192.168.1.10
端口：7
超时：200 ms
重试间隔：100 ms
```

实际值以连接设置窗口为准。

## 8. 真实加工

真实加工前，请确认：

- 已完成路径规划。
- 已执行模拟加工并确认路径正确。
- 模型没有超出加工区域。
- 设备、轴模式、板卡参数、激光器参数正确。
- TCP/串口联锁检查通过。
- 急停、光闸、门禁和加工环境均已就绪。

开始方式：

```text
右上角 -> 开始加工
```

或：

```text
文件 -> 执行任务
```

当前版本开始加工不再弹出开关光或段级反馈确认框。是否开光由右侧“图案参数 -> 激光输出 -> 开光”决定：

- 未勾选：只下发运动轨迹，开光位为关光。
- 已勾选：按路径和延时参数生成开光帧。

数据下发默认连续写入 TCP 双缓冲，结束时统一补齐尾部并等待发送排空。界面进度和路径高亮按加工计划推进，不再依赖段级反馈。

建议首次联调先取消“开光”，确认轨迹正确后再勾选开光。

### 8.1 暂停

右上角：

```text
暂停
```

暂停会停止继续推进加工步骤，保留当前进度。

### 8.2 急停

右上角：

```text
急停
```

急停会停止数据发送并停止 TCP 线程。硬件侧急停仍应使用设备独立急停回路，不应只依赖软件按钮。

### 8.3 刷新至起始数据帧

右上角：

```text
刷新
```

用于回到起始数据帧。真实链路下会尝试发送回归起点数据帧。

## 9. 手动操作

菜单：

```text
手动操作 -> 打开手动操作面板
```

手动操作用于小范围检查振镜方向和坐标映射。界面提供：

- 步长输入，单位 mm。
- X/Y/Z 三行轴向移动按钮。
- STOP 回中按钮。

点击方向按钮后，软件会按当前步长计算目标位置，检查软限位，生成 TCP 数据帧并下发。点击 STOP 回中会把 X/Y/Z 移到各自软限位中心。

说明：

- 手动操作使用当前“交换 y/z 轴”设置。
- spdlog 会记录目标逻辑坐标、实际数据帧通道坐标、编码值、payload 大小和 TCP 反馈。
- 加工运行中会拒绝手动点动和回中，避免与正在发送的加工数据混用。

## 10. 加工平面校正

菜单：

```text
校正 -> 加工平面3x3正方形校正...
```

该功能用于在振镜加工平面上打 3x3 正方形阵列，帮助校正坐标方向、比例和中心位置。默认参数：

- 正方形边长：2 mm。
- 中心间距：5 mm。
- 阵列：3 x 3。
- 中心正方形中心：X=0, Y=0, Z=0。
- 默认开光。

九个正方形的中心位于 X/Y 为 `-5, 0, 5 mm` 的九宫格位置，Z 固定为 0。软件会按当前“交换 y/z 轴”设置生成数据帧；如果你的台子是在交换 Y/Z 后的 XY 加工平面，需先在“设备设置”中勾选“交换 y/z 轴”。

校正图案发送前请确认：

- TCP 主机和端口正确。
- 软限位覆盖 `X/Y = -6 到 6 mm`、`Z=0`。
- 需要真实打标时勾选“开光加工校正图案”。
- 首次建议先不开光观察运动方向。

## 11. 视图操作

### 11.1 3D 视图

常用操作：

- 鼠标选择 Patch。
- 查看模型、纹理和路径。
- 打开或关闭线框、包围盒、坐标系。
- 切换着色模式：默认、法向量、曲率。

菜单：

```text
视图
```

### 11.2 UV 视图

UV 视图用于检查展开、纹理和路径位置。

可开启：

```text
视图 -> UV校验模式
```

校验模式下点击 UV 点，可在 3D 视图中联动高亮对应顶点。

### 11.3 路径颜色

一般规则：

- Mark 加工段：红色路径，已加工时绿色高亮。
- Jump 跳转段：灰色或虚线，已加工时蓝色高亮。

实际颜色以当前主题和视图实现为准。

## 12. 模型偏置与加工区域

菜单：

```text
工具 -> 模型偏置与加工区域
```

用途：

- 显示加工区域黄框。
- 自动居中模型。
- 手动调整模型偏置。
- 重置模型变换。
- 检查模型是否超出加工区域。

默认板卡常用加工区域显示为：

```text
+-32.768 mm
```

如果真实下发前模型超出区域，软件会阻止下发。

## 13. 自适应均匀细分

菜单：

```text
工具 -> 自适应均匀细分
```

用途：

- 对当前模型做均匀细分。
- 提升曲面路径映射和显示的局部精度。
- 细分后可通过 `文件 -> 导出模型` 导出 OBJ/STL。

注意：

- 细分会增加三角形数量和计算压力。
- 如果当前模型三角形数已达到上限，软件可能跳过细分。

## 14. 验证示例流程

菜单：

```text
工具 -> 验证示例流程(0.1mm)
工具 -> 验证示例流程2(圆弧,HUST,0.01mm)

```

用途：

## 15. 命令行模式

入口：

```text
帮助 -> 命令行模式
```

也可以手动启动：

```text
laser_cam_qt.exe --cli
```

常用命令：

```text
help
status
pwd
cd <path>
ls [path]
dir [path]
load_model <path>
parameterize [LSCM|ARAP|AUTHALIC]
import_svg <path> [tile_u] [tile_v]
generate_pattern <strategy> <spacing_mm> [angle_deg]
map_to_xyz
assign_params [curvature]
plan_path
save_job <path>
load_job <path>
estimate_time
exit
quit
```

CLI 路径会按当前工作目录解析。可用 `pwd` 查看当前位置，用 `cd <path>` 切换目录，用 `ls` 或 `dir` 查看目录内容；路径包含空格时请加引号。

示例：

```text
pwd
ls docs
load_model "docs/model1111.STL"
parameterize LSCM
import_svg "docs/HUST.svg" 0.05 0.05
map_to_xyz
assign_params curvature
plan_path
save_job "out/job.json"
```

命令行模式主要用于开发验证和离线任务生成，普通操作建议使用图形界面。

## 16. 日志

菜单：

```text
日志 -> 打开日志
日志 -> 清理当前日志
日志 -> 清理所有日志
```

常见日志位置：

```text
nbcam.log
log/
```

“日志 -> 打开日志”优先打开最近一次 frame 日志。连接流程本身不再生成连接情况日志，路径规划和验证示例生成的日志文件名形如：

```text
frame_<model>_<svg>-<timestamp>.log
```

日志用于排查：

- 软件启动失败。
- 模型加载失败。
- 参数化失败。
- SVG 贴图失败。
- 路径规划失败。
- 通信连接失败。
- 真实下发失败。
- 未处理异常和调用栈。

联调问题建议同时保存：

- `nbcam.log`。
- 最近生成的 `log/*.log`。
- 使用的模型和 SVG 文件。
- 当前设备、板卡、激光器和连接设置截图。

## 17. 测量工具

菜单：

```text
测量
```

功能：

- 计算面积。
- 计算曲面点坐标。
- 计算两点直线距离。
- 计算两点曲面测地线距离。

## 18. 常见问题

### 18.1 软件无法启动

检查：

- 可执行文件目录是否有 Qt/VTK 运行时 DLL。
- 是否存在 `platforms/qwindows.dll`。
- 是否存在 `qt.conf`。
- 是否混用了 Debug 和 Release DLL。
- `nbcam.log` 是否记录异常栈。

### 18.2 打开模型失败

检查：

- 文件路径是否存在。
- 模型格式是否受支持。
- 模型是否损坏。
- 是否含有异常坐标或非三角面。

### 18.3 无法选择 Patch

检查：

- 是否已执行面片聚类。
- 聚类阈值是否过小或过大。
- 模型是否可见。
- 当前是否处于可选取的 3D 视图状态。

### 18.4 参数化失败

检查：

- Patch 是否过小或拓扑异常。
- 模型是否存在破面、自交、重复点。
- 尝试换用 LSCM、ARAP 或 Authalic。
- 重新聚类后再参数化。

### 18.5 应用 SVG 纹理失败

检查：

- Patch 是否已参数化。
- SVG 文件是否存在。
- SVG 是否为空或路径过于复杂。
- 是否包含不受支持的 SVG 特性。

### 18.6 无法执行路径规划

检查：

- 是否已加载模型。
- 是否已执行 Patch 聚类。
- 是否已选中 Patch。
- 选中 Patch 是否已参数化。
- 是否已应用 SVG 纹理到 Patch。
- 填充间距是否过小。
- 轮廓模式是否导致无可加工区域。

### 18.7 模型超出加工区域

使用：

```text
工具 -> 模型偏置与加工区域
```

可查看黄框加工区域，尝试自动居中或手动调整偏置。真实下发前软件会阻止超出加工区域的任务。

### 18.8 TCP 连接失败

检查：

- 板卡 IP 是否正确。
- 端口是否正确。
- 网线和防火墙。
- 板卡是否处于可连接状态。
- TCP 超时和重试间隔是否过小。
- 是否有其他程序占用连接。

### 18.9 串口联锁失败

检查：

- 串口号是否正确。
- 波特率是否正确。
- 串口是否被其他软件占用。
- 激光器是否上电并处于可通信状态。
- USB 转串口驱动是否正常。

### 18.10 RTC-6 无法执行

检查：

- 是否选择 RTC-6 板卡。
- `RTC6DLLx64.dll` 是否在可执行文件目录。
- RTC-6 驱动和硬件是否正常。
- 卡号是否正确。
- 程序文件路径是否有效。
- 软限位和 `units_per_mm` 是否设置正确。

### 18.11 模拟加工后半段变慢

路径数量非常大时，显示层压力会增加。可尝试：

- 使用 Release 版本。
- 关闭 UV 视图。
- 增大填充间距，减少路径点数量。
- 先只规划小范围 Patch 验证参数。
- 避免在低性能显卡或远程桌面环境中显示超大路径。

### 18.12 激光器参数导入后不完整

PDF 文件不会自动解析所有参数。导入 PDF 后，请人工核对并填写：

- 功率范围。
- 频率范围。
- 控制方式。
- 串口参数。

### 18.13 振镜运动方向不对

检查：

- `设备与板卡 -> 设备设置` 中是否需要勾选“交换 y/z 轴”。
- 手动操作 X/Y/Z 三个方向是否与台面实际方向一致。
- 校正 3x3 正方形是否位于预期中心和方向。
- 板卡设置中的零点偏置和软限位是否正确。
- 当前模型是否经过额外变换或偏置。

## 19. 安全注意事项

真实加工前必须确认：

1. 硬件急停可用。
2. 门禁、光闸、联锁回路可用。
3. 激光器处于受控状态。
4. 加工区域无人和无反射风险物。
5. 模型路径已模拟确认。
6. 板卡和激光器参数已核对。
7. 首次联调取消“开光”，只观察运动轨迹。
8. 真实出光必须由具备设备权限的人员确认。

软件按钮不能替代硬件安全回路。急停、门禁、光闸和联锁应以硬件系统为准。

## 20. 推荐操作顺序

首次使用建议按以下顺序：

1. 打开软件。
2. `设备与板卡 -> 设备设置`，选择默认板卡或 RTC-6，选择三轴/五轴，并按实际设备确认是否勾选“交换 y/z 轴”。
3. `设备与板卡 -> 板卡设置`，确认偏置、软限位和时序参数。
4. `设备与板卡 -> 激光器设置`，确认激光器参数。
5. `连接 -> 连接`，填写 TCP/串口参数。
6. 使用通信测试确认链路。
7. `文件 -> 打开模型`。
8. `工具 -> 模型偏置与加工区域`，确认模型位于加工区域内。
9. `工具 -> 面片聚类`。
10. 在 3D 视图选择 Patch。
11. `工具 -> 参数化选中Patch`。
12. `工具 -> 应用SVG纹理到Patch`。
13. 设置右侧加工参数。
14. `工具 -> 路径规划 -> 执行路径规划`。
15. 打开路径预览检查路径段。
16. 使用右上角“模拟加工”。
17. 使用手动操作和校正图案确认方向、中心和比例。
18. 真实加工前再次确认硬件安全。
19. 右侧取消“开光”后试跑。
20. 确认无误后勾选“开光”真实出光。
