# 课题中期进展报告

课题名称：基于纹理映射的复杂曲面激光标刻/加工软件系统研究与实现  
软件系统：NBCAM  
报告日期：2026-05-07  
阶段性质：中期进展总结

## 一、课题研究目标

本课题面向复杂曲面激光标刻与精密加工应用，研究并实现一套从三维模型导入、曲面区域识别、参数化展开、二维图案映射、曲面路径生成、加工路径预览到板卡数据帧下发的软件系统。系统目标是在复杂曲面上实现可视化、可规划、可验证、可联调的激光加工流程，为后续曲面变参数加工、曲面均匀加工和多轴协同加工提供工程基础。

课题的主要研究内容包括：

1. 复杂曲面三角网格模型的导入、显示和加工区域管理。
2. 曲面 Patch 聚类与局部参数化方法。
3. SVG 图案到曲面 Patch 的纹理映射和路径生成。
4. UV 路径到三维曲面路径的映射。
5. 激光加工任务数据模型和路径段组织方式。
6. 默认板卡与 RTC-6 板卡的数据下发和联调流程。
7. 曲面加工均匀性相关的工艺参数模型和验证方案。

## 二、总体技术路线

系统采用“Qt 图形界面 + 核心算法库 + 执行器通信层”的分层架构。整体流程如下：

```text
三维模型 STL/OBJ
  -> 网格加载与预处理
  -> Patch 聚类与选择
  -> Patch 参数化
  -> SVG 图案贴附与 UV 映射
  -> UV 路径生成
  -> UV 到 XYZ 曲面路径映射
  -> LaserJob 任务组织
  -> 路径预览与模拟加工
  -> TCP/RTC-6 数据帧下发
```

软件主要采用 C++17 开发，图形界面基于 Qt6，三维显示基于 VTK，几何计算和参数化能力依赖 CGAL，任务序列化采用 JSON，日志系统采用 spdlog，SVG 解析采用 nanosvg。

### 3.3 曲面 Patch 处理与参数化

当前系统可以对导入的三角网格模型进行基于法向和二面角阈值的 Patch 聚类。用户可在 3D 视图中选择指定 Patch，并对局部 Patch 执行参数化。

已实现能力：

- 基于二面角阈值的 Patch 聚类。
- Patch 面积、法向和三角形数量统计。
- LSCM/ARAP/Authalic 参数化算法接入。
- Patch 局部参数化结果回写到 UV 视图。
- UV 校验模式和 3D/UV 联动查看。

该部分为后续复杂曲面局部加工路径生成提供了基础。

### 3.4 SVG 图案映射与路径生成

系统已实现 SVG 图案加载、贴图参数编辑和基于 SVG 区域的路径生成。当前支持的加工策略包括：

- 轴向往返填充。
- 轴向往返填充（圆弧）。
- 回型填充（内到外）。
- 回型填充（外到内）。
- 轮廓偏置回型。
- Z 轴分层填充。
- 只加工轮廓、不加工轮廓、加工全部等轮廓模式。

在路径生成阶段，软件能够结合 SVG Alpha 区域和 Patch UV 域进行裁剪，使路径尽量限制在目标图案区域内。生成后的路径可以在 3D 视图和 UV 视图中显示，并可通过路径预览窗口查看每个路径段。

### 3.5 LaserJob 数据模型与路径预览

系统已建立统一的 `LaserJob` 任务模型，用于描述激光加工任务。该模型包含：

- 任务元数据。
- 坐标系和模型到机床坐标的变换信息。
- 参数化信息。
- 默认加工参数。
- 路径段集合。
- 路径点集合。
- 点级和段级工艺参数覆盖。

路径预览窗口可显示段 ID、路径类型、激光状态、起终点坐标、点数和长度。用户可以按段筛选、聚焦、导出 CSV，并在 3D/UV 视图中同步高亮选中的路径段。

### 3.6 板卡执行与通信联调

当前已完成默认板卡链路的基础下发能力，核心链路为：

```text
LaserJob
  -> BoardExecutor / MainWindow 批次构建
  -> DataBuffer 双缓冲
  -> TcpSocket 发送线程
  -> TCP 发送至板卡
```

默认板卡采用 16 字节固定数据帧，每帧由 8 个小端 `uint16_t` 组成：

```text
B, A, Z, Y, X, OPCODE, arg7, arg8
```

主要帧类型包括：

- Begin 帧。
- Mark 点帧。
- Jump 点帧。
- End 帧。
- 频率设置帧。
- 功率设置帧。

已完成的联调改进包括：

- TCP 连接配置、超时和重试配置。
- 真实下发前联锁检查。
- 逐段反馈和连续下发两种模式。
- 发送缓冲区排空等待。
- 修复小批次数据被扩展为整块 1.6 MB 缓冲区发送的问题，使段级发送更符合实际有效数据长度。
- 增加排空失败时的发送队列状态和 TCP 最近反馈信息，便于定位连接或缓冲区问题。

RTC-6 链路已建立框架，可配置卡号、编码系数、默认扫描速度、跳转速度和程序文件。当前 RTC-6 仍需要硬件、驱动和 DLL 环境进一步联调。

## 五、关键问题与解决情况

### 5.1 曲面路径生成与 Patch 参数化问题

复杂曲面上直接生成二维图案路径容易出现形变和越界问题。本阶段采用 Patch 局部参数化和 SVG Alpha 区域裁剪的方法，将二维图案路径限制在选中 Patch 的有效区域内，并通过 UV 到 XYZ 映射生成曲面路径。

### 5.2 大规模路径显示性能问题

路径数量较大时，3D 和 UV 视图的高亮更新会带来性能压力。当前已采用增量高亮方式：

- 模拟加工时表格只选中当前段。
- 3D 视图累计显示已加工段。
- UV 视图采用叠加缓存进行已加工段绘制。

### 5.3 TCP 缓冲区排空失败问题

实际联调中出现过“数据帧未在超时时间内排空”的问题。经排查，原因之一是 `forceFill()` 记录实际有效数据长度后，后续逻辑又将长度覆盖为整块 1.6 MB 缓冲区，导致小 segment 也按 1.6 MB 发送。

目前已修复：

- 保留实际有效字节数。
- 空缓冲不再入队发送。
- 排空失败时输出队列状态和 TCP 反馈。

### 5.4 曲面变参数加工暂缓实装

课题中涉及曲面变参数加工，即根据曲率、光斑搭接率和单位面积能量调整功率、速度。根据当前阶段安排，该功能暂不直接实装到软件执行链路，而是先完成数学模型和验证方案，避免未经实验标定的模型直接影响真实加工。

当前已保留理论文档：

```text
reference/曲面变参数加工数学模型.md
```

后续将通过离线仿真和实物样件对比验证模型有效性。

## 八、下一阶段实验验证方案

针对曲面加工均匀性，建议按以下方案进行：

1. 平面样件上标定基准功率和速度。
2. 已知半径球面或圆柱面上进行固定参数加工。
3. 离线计算曲率和建议变参数，不直接下发。
4. 生成搭接率和单位面积能量分布 CSV。
5. 观察模型是否能降低高曲率区域的理论能量波动。
6. 经安全确认后，再进行小范围样件试加工。
7. 比较槽宽、槽深、表面粗糙度和烧蚀边缘。
8. 采用变异系数评价均匀性：

```text
CV_width = std(width) / mean(width)
CV_depth = std(depth) / mean(depth)
```

若变参数方案能显著降低 `CV_width` 和 `CV_depth`，说明模型具备进一步实装价值。
